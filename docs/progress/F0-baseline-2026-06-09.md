# F0 Old-Baseline — UC12-min 최소 부팅 (2026-06-09)

> ⚠️ **이것은 코드 도출 baseline spec** (old-naia-os 소스에서 거동 특성화). **라이브 trace 확정 = 루크 머신 구동 필요**(Tauri 빌드). 로컬·외부키X라 코드에서 대부분 결정적으로 도출됨 — 라이브는 timing·실제 I/O·edge 확정용.

## 소스 경로 (old-naia-os)

- **부팅 게이트(정정 R1)**: `App.tsx` `showAdkSetup` = **localStorage ADK-path 유무**(`adk-store.ts isAdkInitialized/getAdkPath`)로 결정 — *naia-settings 존재가 아님*. path 없으면 디스크에 기존 ADK 있어도 `<AdkSetupScreen>`(L734). setup 에서 `workspace_detect_adk_root`로 **defaultPath 자동 *제시*만**(L52·58) — *자동 setAdkPath/완료 없음, 사용자가 load/new/login 확정 필요*. 즉 'first run' 아니라 **'local ADK-path bootstrap missing'**. installed 패널 `loadInstalledPanels()`(L314)=실패해도 **non-fatal**(예외 먹고 진행).
- **workspace root 설정**: `workspace.rs::workspace_set_root`(L510) — ① dir 검증(`is_dir` 아니면 Err) ② `dunce::canonicalize`(경로 정규화 — symlink/mount/대소문자) ③ inaccessible 이면 Err ④ `WORKSPACE_ROOT_OVERRIDE` mutex 저장. → **FR-F1.4 canonical workspace root 와 정합**(이미 정규화 함).
- **키 optional**: App.tsx:405 `if (!cfg?.naiaKey) return` — naiaKey 없어도 부팅 진행, `auth_update`만 스킵. → **외부키 없는 부팅 경로 실재 확인**(FR-F0 충족).
- control-plane 커맨드: `init_naia_settings`·`check_naia_settings`·`read_naia_config`·`workspace_detect_adk_root`.

## Golden reference 거동 (코드 도출)

| 입력 상태 | 거동 (출력/상태, 코드상) | disposition(앱 레벨) |
|---|---|---|
| ADK-path(localStorage) 없음 | `AdkSetupScreen` + `workspace_detect_adk_root` defaultPath 자동제시(사용자 확정 필요, 자동완료 X) | **contain**(부트스트랩 안내) |
| ADK-path 있음·키 없음 | `showAdkSetup` 우회, auth_update 스킵. **단 `!isOnboardingComplete()`면 main shell(TitleBar/Avatar/AiControlBar/ChatPanel)은 유지되고 우측 main content만 `OnboardingWizard`로 대체** | 정상(외부키X; onboarding 게이트 별도) |
| ADK-path 있음·키 있음 | main + auth_update 전송 | 정상(+외부) |
| workspace root ≠ dir / inaccessible | 백엔드 `workspace_set_root`→`Err`, **앱은 WORKSPACE_ROOT fallback + stale config clear + workspaceReady=true 로 진행** | **contain + fallback**(앱 레벨; *block 아님* — 정정 R1) |

## 상태 전이 (hidden state)
- `WORKSPACE_ROOT_OVERRIDE` mutex(canonical path) · naia settings/config 파일 · installed 패널 목록. (측정 간 리셋 대상.)

## 오류 분류 (F0 taxonomy 적용 — 정정 R1)
- workspace root invalid/inaccessible = 백엔드 `Err`이나 **앱 레벨 disposition = contain + fallback**(WORKSPACE_ROOT fallback·stale clear·진행). *block 아님.* ⚠️ FR-F0 요구사항(integrity-critical=fail-closed)과 **old 실제(contain+fallback)가 다름** → 이식 시 disposition 유지 vs 강화(fail-closed) = 결정 대상(baseline은 old 실제 기록).
- ADK-path 부재 = **bootstrap missing → contain**(setup + defaultPath 제시, 사용자 확정).
- naiaKey 부재 = degradation 아님(정상 무키 모드).

## 커버리지 manifest (F0)
- **accepted**(이식 대상): 부팅 게이트(`isAdkInitialized`/`getAdkPath`) · `workspace_detect_adk_root`(defaultPath 제시) · `init_naia_settings`(load/new 경로) · `workspace_set_root`(canonical, 실패=fallback) · `read_naia_config` · `loadInstalledPanels`(non-fatal). (`check_naia_settings`=현 부팅경로 실사용 약함 → 제외/확인.)
- **deferred**: 외부 auth(naiaKey·auth_update) = V1 전 UC12 전체에서(외부 의존).

- 미분류 = 0.
