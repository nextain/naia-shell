# F0 Old-Baseline — UC12-min 최소 부팅 (2026-06-09)

> 06 실행 1단계 = F0 Old-Baseline 측정. **범위**: UC12-min(외부키 없는 workspace 최소 부팅, control-plane init).
> ⚠️ **이것은 코드 도출 baseline spec** (old-naia-os 소스에서 거동 특성화). **라이브 trace 확정 = 루크 머신 구동 필요**(Tauri 빌드). 로컬·외부키X라 코드에서 대부분 결정적으로 도출됨 — 라이브는 timing·실제 I/O·edge 확정용.

## 소스 경로 (old-naia-os)

- **부팅 게이트(정정 R1)**: `App.tsx` `showAdkSetup` = **localStorage ADK-path 유무**(`adk-store.ts isAdkInitialized/getAdkPath`)로 결정 — *naia-settings 존재가 아님*. path 없으면 디스크에 기존 ADK 있어도 `<AdkSetupScreen>`(L734). setup 에서 `workspace_detect_adk_root`로 **defaultPath 자동 *제시*만**(L52·58) — 완료 시 **`copy_bundled_assets`(항상 호출 — 현 구현은 실제 복사 아님, `asset://` scope 확장만 후 종료)**, `new` 경로는 **`inspect_adk_dir`(선행)** + 조건부 **`clone_naia_adk`/`delete_naia_adk`**(L184) 부트스트랩 실행 — *자동 setAdkPath/완료 없음, 사용자가 **load/new** 확정 필요(login 카드는 `disabled` — 확정 경로 아님)*. 즉 'first run' 아니라 **'local ADK-path bootstrap missing'**. installed 패널 `loadInstalledPanels()`(L314)→ Tauri `panel_list_installed` invoke, 실패해도 **non-fatal**(예외 먹고 진행).
- **workspace root 설정**: `workspace.rs::workspace_set_root`(L510) — ① dir 검증(`is_dir` 아니면 Err) ② `dunce::canonicalize`(경로 정규화 — symlink/mount/대소문자) ③ inaccessible 이면 Err ④ `WORKSPACE_ROOT_OVERRIDE` mutex 저장. → **FR-F1.4 canonical workspace root 와 정합**(이미 정규화 함).
- **키 optional**: App.tsx:405 `if (!cfg?.naiaKey) return` — naiaKey 없어도 부팅 진행, `auth_update`만 스킵. → **외부키 없는 부팅 경로 실재 확인**(FR-F0 충족).
- control-plane 커맨드(부팅경로 실호출): `init_naia_settings`(load/new) · `read_naia_config` · `write_naia_config`(아래 조건부) · `write_naia_path_cache`(`setAdkPath`→ localStorage **+ `~/.naia/adk-path`** 갱신 — agent startup path discoverability) · `workspace_detect_adk_root` · **`store_startup_message`** + **`send_to_agent_command`**(부팅 시 `notify_config`·`creds_update`를 **둘 다** — startup cache 저장(재시작 replay) **및** 라이브 전송; `auth_update`만 `naiaKey` 있을 때 조건부. ⇒ 무키 UC12-min 에서도 notify_config/creds_update 전송됨) · **`workspace_start_watch`**(watcher 재시작 — 2경로: `workspace_set_root` 성공 직후 **+** workspace 패널 `onActivate`). (`check_naia_settings`=부팅경로 **미호출**, 팬텀 — 이식 제외, R5.)
- **`write_naia_config` 조건부(R5)**: App.tsx mount `syncConfigToFile()` 호출되나 실제 파일 쓰기는 `loadConfig()` 존재 **&&** `getAdkPath()` 존재 시에만(`adk-store.ts::writeNaiaConfig`). **ADK-path/설정 없으면 no-op** — F0 무키·초기 케이스에선 흔히 no-op. ⚠️ raw write-back 아님: **`stripForAgent()`로 secret/UI 필드 제거** 후 기록 + `buildNaiaConfigEnv()` 파생 env 키 병합(R7) — 이식 시 secret-strip 변환 누락 금지(보안).

## Golden reference 거동 (코드 도출)

| 입력 상태 | 거동 (출력/상태, 코드상) | disposition(앱 레벨) |
|---|---|---|
| ADK-path(localStorage) 없음 | `AdkSetupScreen` + `workspace_detect_adk_root` defaultPath 자동제시(사용자 load/new 확정 필요; login disabled; 자동완료 X) | **contain**(부트스트랩 안내) |
| ADK-path 있음·키 없음 | `showAdkSetup` 우회, auth_update 스킵. **단 `!isOnboardingComplete()`면 main shell(TitleBar/Avatar/AiControlBar/ChatPanel)은 유지되고 우측 main content만 `OnboardingWizard`로 대체** | 정상(외부키X; onboarding 게이트 별도) |
| ADK-path 있음·키 있음 | main + auth_update 전송 | 정상(+외부) |
| workspace root ≠ dir / inaccessible | 백엔드 `workspace_set_root`→`Err`, **`WorkspaceCenterPanel`(content-panel) 마운트 후** WORKSPACE_ROOT fallback + stale config clear + workspaceReady=true. *onboarding 중엔 content-panel 미렌더라 미발동* | **contain + fallback**(content 마운트 후; block 아님 — R1/R4) |

## 상태 전이 (hidden state)
- `WORKSPACE_ROOT_OVERRIDE` mutex(canonical path) · naia settings/config 파일 · installed 패널 목록. (측정 간 리셋 대상.)

## 오류 분류 (F0 taxonomy 적용 — 정정 R1)
- workspace root invalid/inaccessible = 백엔드 `Err`이나 **앱 레벨 disposition = contain + fallback**(WorkspaceCenterPanel 마운트 후 WORKSPACE_ROOT fallback·stale clear·진행; onboarding 중 미발동). *block 아님.* ⚠️ FR-F0 요구사항(integrity-critical=fail-closed)과 **old 실제(contain+fallback)가 다름** → 이식 시 disposition 유지 vs 강화(fail-closed) = 결정 대상(baseline은 old 실제 기록).
- ADK-path 부재 = **bootstrap missing → contain**(setup + defaultPath 제시, 사용자 확정).
- naiaKey 부재 = degradation 아님(정상 무키 모드).

## 커버리지 manifest (F0)
- **accepted**(이식 대상): 부팅 게이트(`isAdkInitialized`/`getAdkPath`) · `workspace_detect_adk_root`(defaultPath 제시) · `init_naia_settings`(load/new 경로) · `workspace_set_root`(canonical, 실패=fallback) · `read_naia_config` · `write_naia_config`(조건부 write-back: loadConfig&&getAdkPath, 아니면 no-op) · **`write_naia_path_cache`**(`~/.naia/adk-path`) · **`store_startup_message`**(replay cache) + **`send_to_agent_command`**(notify_config/creds_update 라이브; auth_update=키 조건부) · **`workspace_start_watch`**(set_root 후 + 패널 onActivate) · setup 부트스트랩 **`copy_bundled_assets`(asset:// scope 확장)·`inspect_adk_dir`·조건부 `clone_naia_adk`/`delete_naia_adk`** · `loadInstalledPanels`(non-fatal). (`check_naia_settings`=부팅경로 미호출 팬텀 → **제외 확정**.)
- **deferred**: 외부 auth(naiaKey·auth_update) = V1 전 UC12 전체에서(외부 의존).
- **rejected**: 없음.
- 미분류 = 0.
