## S01 온보딩 — 8단계 순서 + 단계↔필드 매핑 (golden, per-step 측정)
`welcome → agentName → userName → speechStyle → character → background → provider → complete`
각 단계 = naia 1인칭 챗 프롬프트(`stepChat`) + 입력/선택 UI. 단계가 set 하는 상태(완료 시 config 로):
| 단계 | set 하는 필드(latestRef → config) |
|------|-----------------------------------|
| welcome | (없음 — 안내) |
| agentName | `agentName` |
| userName | `userName` (+ `honorific`) |
| speechStyle | `speechStyle` (+ `extraPersona`) |
| character | `selectedVrm` → `vrmModel` |
| background | `selectedBg` → `background` (naia-settings/background 자산서 로드, 기본=space) |
| provider | `apiKey` \| `naiaKey`(Naia OAuth callback→`naiaLoginDone`) ; 로그인 시 `memoryEmbeddingProvider`/`memoryLlmProvider` 자동 "naia" |
| complete | 위 전부 `saveConfig` + `onboardingComplete=true` + `syncToGateway` |
- VRM/배경 목록 = `listNaiaAssets("vrm-files"|"background")` (= `{adkPath}/naia-settings/` 자산). 기본 VRM=목록[0], 기본 배경=space.
- naia 계정 분기: provider 단계서 OAuth callback(`naia-remote-key` 저장 + `sendAuthUpdate`) → complete.

## S02/S03 설정 — config 산출 표면 (golden)
`AppConfig`(config.ts) 주요 필드 = 온보딩/설정이 produce 하는 계약:
- 신원/표현: `agentName`, `userName`, `speechStyle`, `vrmModel`(character), `background`, `locale`, `theme`
- provider/모델: `provider`(nextain·anthropic·openai·gemini·ollama·vllm·glm·claude-code-cli), `model`, `apiKey`|`naiaKey`, `naiaGatewayUrl`
- 음성/STT/TTS: `voiceMode`/`sttProvider`/`ttsProvider` 등
- 기억: `memoryEmbeddingProvider`/`memoryLlmProvider` 등
- 게이트: `onboardingComplete`, `enableTools`
- 영속: localStorage "naia-config" (+ 비밀키는 secure-store). naia-settings/config.json = NAIA_MAIN_* 변환본(buildNaiaConfigEnv).
