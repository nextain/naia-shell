# F0 graft 부팅 준비 — 루크 머신 실행 안내 (2026-06-09)

> 목적 = **"동작" 첫 증명**. new-naia-os 의 F0 `ControlPlaneBoot`(async, codex 코드리뷰 통과)를 **옛 앱(old-naia-os, 실제 Tauri 백엔드+shell 보유)** 에 graft 해서, 새 core 가 *실제 백엔드*로 계산한 부팅 결정이 **옛 앱의 실제 결정과 같은지**(Old-Baseline 등가 = drift-gate) 확인한다.
> ⚠️ **1차 스모크 = 읽기 전용**(`graftBootDecisionSmoke`) — set/save 안 함, 루크 설정 안 건드림. 안전.
> ⚠️ old-naia-os 는 frozen — graft 는 **임시 dev 패치(worktree 권장)**, 커밋 금지.

## 왜 이게 핵심인가
계약테스트 54/54 통과·codex/gemini/GLM 리뷰 통과 = **로직**만 검증. "실제로 도나"는 **미증명**. 이 graft 가 처음으로 *실제 백엔드*에 붙여 new core 의 결정을 old 실제와 비교한다 — paper→실증의 첫 관문. (async 갭도 실배선 시도로 잡혔듯, 여기서 또 뭔가 드러나면 그게 소득.)

## 절차 (루크 머신)

### 1. new-naia-os 빌드
```bash
cd projects/new-naia/new-naia-os
pnpm install && pnpm build   # → dist/main/**
```

### 2. old-naia-os dev worktree (frozen 보호)
```bash
cd projects/new-naia/old-naia-os
git worktree add ../old-naia-os-graft HEAD   # 임시 작업본
```

### 3. graft 코드 (old worktree 의 shell/src/App.tsx mount effect 에 추가)
```ts
// ⚠️ 임시 graft — 커밋 금지. 빌드 후 dist 를 상대경로로 import (또는 새 core 소스를 shell/src/_graft/ 에 복사).
import { graftBootDecisionSmoke } from "<new-naia-os>/dist/main/app/control/graft-smoke.js";
import { invoke } from "@tauri-apps/api/core";
import { loadConfig, saveConfig, loadConfigWithSecrets, isOnboardingComplete } from "./lib/config";
import { getAdkPath, setAdkPath } from "./lib/adk-store";

// App mount 시 1회:
graftBootDecisionSmoke({
  invoke,
  loadConfig, saveConfig, loadConfigWithSecrets,
  getAdkPath, setAdkPath, isOnboardingComplete,
}).then((r) => console.log("[F0-GRAFT]", r));
```

### 4. 빌드+실행 → 콘솔 확인
```bash
cd ../old-naia-os-graft/shell && pnpm run tauri:dev
```
DevTools 콘솔에서 `[F0-GRAFT]` 로그 확인:
- `match: true` → ✅ **새 core 가 실제 백엔드로 옛 앱과 같은 부팅 결정.** Old-Baseline 등가 1차 입증.
- `match: false` → ⚠️ drift. `newDecision` vs `oldExpectedDecision` 비교해 원인 조사(= 코드-도출 baseline 이 실제와 다른 지점 = 소득).

### 5. (다음) 쓰기 경로 스모크
읽기 등가 확인 후 → `wireControlPlaneLive(deps).boot()` 전체(2b·merge·detectRoot 포함)를 dev 에서 실행해 실제 부팅 구동. set/save 포함이라 **dev workspace 에서만**.

## 정리
- 1차(읽기) `match: true` = "새 core 가 실제로 옛 앱과 같게 판단한다" 입증 → 방법+배선+baseline 신뢰 ↑.
- 이게 **검증 테스트(P02 1단계 Old-Baseline 등가 = drift-gate)의 첫 실물**. 계약테스트(2단계)는 이미 54/54.
- 정리 = worktree 제거 `git worktree remove ../old-naia-os-graft`.
