> 06 실행 2단계 = F0 **포트 시그니처 계약**(STRUCTURE.md step-2 deferred 항목). 입력 = 동결된 [F0-baseline](./F0-baseline-2026-06-09.md)(코드레벨 CLOSED). **이 문서 = 계약(인터페이스)이지 구현 아님.** 헥사고날 레이어 규칙(STRUCTURE.md 171~297) 적용: `adapters→ports→domain`·`app→ports`·단일 composition root·protocol transport-neutral.

## 1. domain/ (순수 값객체·규칙, import 0)

| 값객체 | 규칙(불변식) | baseline 근거 |
|---|---|---|
| `CanonicalRoot` | **검증된 결과값**(canonical path). ⚠️ canonicalize/`is_dir` = **filesystem I/O라 domain 아님** — `WorkspacePort.setRoot` 어댑터/백엔드가 수행하고 `CanonicalRoot` 또는 `Err` 반환(C-R1 MAJOR 정정). domain 은 결과 값만 모델(I/O 0). | `workspace_set_root`(workspace.rs) |
| `NaiaConfig` | 필드 분류 규칙: `secret`(키/크레덴셜)·`ui`(표시전용)·`agent`(에이전트 노출 가능). **`forAgent()` = secret+ui 제거** 후 agent-facing 투영 산출(순수 변환). | `write_naia_config`의 `stripForAgent()` |
| `AdkPath` | 존재/유효(localStorage 값) — `Present(path)` \| `Absent`. | `isAdkInitialized`/`getAdkPath` |
| `BootDecision` | 순수 결정: `(adkPathPresent, onboardingComplete)` → `SetupRequired`(path 부재) \| `OnboardingOverlay`(path 有·onboarding 미완 → main shell 유지+우측 content 대체) \| `Main`(둘 다). **입력 둘 다 포트로 조회**(직접 read 금지 — §2 `BootStatePort`). | App.tsx showAdkSetup/onboarding 게이트(codex R1·R3) |
| `StartupMessage` | 종류 `NotifyConfig`\|`CredsUpdate`\|`AuthUpdate`. **발신 조건 규칙**: NotifyConfig/CredsUpdate = `config 존재`(loadConfigWithSecrets non-null) 필요; AuthUpdate = 추가로 `naiaKey 존재` 필요. | App.tsx initAuth(codex R10·R12) |

> domain 은 어떤 포트/어댑터/transport/filesystem 도 모름. 위 규칙은 전부 입력→출력 순수 함수. (canonicalize·is_dir 같은 I/O 는 domain 금지 — 포트 뒤로.)

## 2. ports/ (인터페이스 = 계약, domain 만 의존)

전부 **driven 포트**(app 이 *사용*). 시그니처 = 언어중립.

```
# ports/protocol  — transport-neutral 의미 payload (직렬화/프레이밍 누출 금지)
StartupMessagePayload  = { kind: NotifyConfig|CredsUpdate|AuthUpdate, body: <semantic struct> }
PanelDescriptor        = { id, title, ... }   # listInstalled 결과 의미형

# ports/  — driven 인터페이스
ConfigPort:
    read(adkPath: AdkPath): NaiaConfig | null            # 없으면 null (config.ts 295)
    write(adkPath: AdkPath, agentView: NaiaConfig.forAgent): void
                                                          # ⚠️ 독립 debounced write-back (App.tsx 350) — startup send 와 순서 결속 없음(C-R1 MED)
BootStatePort:                                            # 부팅 게이트 상태 read/write(직접 storage 접근 금지, C-R1·C-R4 MAJOR)
    mergeFromFile(config: NaiaConfig): void               # ⚠️ **부팅 게이트(§3-A step3) 전용** readNaiaConfig→saveConfig(merged). setup 엔 쓰지 말 것(C-R7)
    isOnboardingComplete(): bool                          # baseline isOnboardingComplete()=loadConfig 읽음(config.ts 555)
    loadLocalConfig(): NaiaConfig | null                  # ⚠️ **plain loadConfig()** — secrets/keychain I/O 없음. 2b 정합화·게이트용(무키 경계 준수, C-R12)
    loadLocalConfigWithSecrets(): NaiaConfig | null        # ⚠️ loadConfigWithSecrets() — **initAuth(§3-D) 전용**(keychain I/O, secrets 경로). 2b/게이트엔 쓰지 말 것(C-R12)
    replaceLocalConfig(config: NaiaConfig): void          # ⚠️ setup: 기존 보존 X, **통째 교체**(load) (C-R7 MAJOR; AdkSetupScreen 285)
    resetLocalConfig(): void                              # ⚠️ setup(use-existing/new/recreate): local state 비우고 최소 재작성(먼저) (C-R7; 42·232·265)
    setWorkspaceRoot(path): void                          # setup 종료 전 저장 (C-R4)
    clearWorkspaceRoot(): void                            # ⚠️ workspace fallback 시 저장된 root 삭제 → 다음 부팅 반복실패 방지(C-R7 MED; WorkspaceCenterPanel 416)
    markOnboardingComplete(): void                        # 완료 표시(모드별 조건 §3-C) — app 이 storage 직접 X
AdkPathPort:
    get(): AdkPath                                        # Present|Absent
    set(path): void                                       # localStorage + path 캐시(홈 디렉터리 .naia 하위) 둘 다
    detectRoot(): AdkPath | null                          # 자동 *제시*만(자동완료 X)
WorkspacePort:                                            # ⚠️ boot 공통 아님 — WorkspaceCenterPanel 트리거(§3-B, C-R1 MED)
    setRoot(root: rawPath): Result<CanonicalRoot, Err>    # canonicalize/is_dir = 어댑터/백엔드. Err = 호출자 contain+fallback. **패널 마운트 effect**(활성화와 분리, C-R8)
    startWatch(): void                                    # ⚠️ **인자 없음** — 패널 onActivate (index.tsx 177, C-R8)
    stopWatch(): void                                     # 패널 onDeactivate (workspace_stop_watch, C-R2 MED)
StartupMessagePort:                                       # initAuth 경로(§3 step 4)
    store(msg: StartupMessagePayload): void               # replay cache (재시작용)
    send(msg: StartupMessagePayload): void                # 라이브 전송
PanelInventoryPort:
    listInstalled(): PanelDescriptor[]                    # ⚠️ setup 게이트 *이전* 호출, 실패=non-fatal contain (C-R1 MAJOR)
CredentialStorePort:   # 【DEFERRED — 외부키 영역, 무키 UC12-min 엔 no-op】 C-R3 MAJOR
    writeAgentKey(envKey, value): void                    # 온보딩 완료 시 naiaKey/apiKey → OS keychain (config.json write 와 별도). 실패=non-fatal(creds_update IPC 가 세션 처리)
AdkSetupPort:                                              # setup 분기(path 부재) — new/load/use-existing/recreate
    initSettings(adkPath): void                           # init_naia_settings(adkPath) — mode 무관 단일 호출(C-R10). 순서 = §3-C
    copyBundledAssets(adkPath): void                      # copy_bundled_assets(adkPath) — path 인자(C-R4). 현 구현=asset:// scope 확장(실제 복사 X)
    inspectAdkDir(path): AdkDirState                      # new/recreate 선행
    cloneAdk(path): void / deleteAdk(path): void          # 조건부(new/recreate)
```

> ⚠️ 불변식(STRUCTURE.md R2): `StartupMessagePayload`·`PanelDescriptor` 에 wire-framing(JSON 라인·길이 prefix·gRPC) 누출 금지 — 직렬화는 transport adapter 만. stdio→gRPC = adapter 교체로 끝.

## 3. app/control/ (포트 사용 = 부팅 오케스트레이션, 인지 0)

`ControlPlaneBoot` — 포트만 사용(adapter 직접 import 금지). baseline 순서를 조건부로 재현. **순서 결속은 baseline 이 실제 결속한 것만**(과결속=drift, C-R1).

> ⚠️ **비동기 렌더-타이밍 비결속(C-R9 바운드)**: 구 shell 의 React effect 경쟁 조건 — ① `showAdkSetup` 이 첫 렌더에 `!isAdkInitialized()` 로 고정되어 이후 `setAdkPath` 가 이번 렌더의 게이트를 안 내림(2b 의 정합화는 *다음 평가용 persist* 일 뿐 현재 게이트 불변) ② `readNaiaConfig` 병합이 비동기라 onboarding 초기 판정이 병합 전 localStorage 기준(파일 `onboardingComplete:true` 여도 일시 overlay flash 가능) — 은 **계약에 결속하지 않는다.** 이식 = 언어/런타임 중립(§6)이고 깨끗한 결정적 부팅이 목표이지 race condition 복제가 아님. 이 타이밍 아티팩트는 라이브 trace/통합 단계에서 *관측*만 하고 계약 테스트로 고정하지 않는다.

### A. boot() — 전역 부팅 (App mount)

```
boot():
  1. try PanelInventoryPort.listInstalled() catch → non-fatal   # ⚠️ 게이트 *이전*, 항상(SetupRequired 포함). C-R1 MAJOR
  2. adk = AdkPathPort.get()
  2b. # persisted state normalization (C-R8 MED): adkPath ↔ config.workspaceRoot 양방향 동기화 — **다음 평가용 persist**(현재 게이트 불변, C-R9)
      cfgRoot = BootStatePort.loadLocalConfig()?.workspaceRoot
      if cfgRoot and cfgRoot != adk.path: AdkPathPort.set(cfgRoot)    # ⚠️ config.workspaceRoot 권위적 — 부재뿐 아니라 stale 교정도(App.tsx 316, C-R10)
      elif adk.present and not cfgRoot: BootStatePort.setWorkspaceRoot(adk.path)  # adkPath 만 있으면 config 채움(317)
  3. if adk.present:                                             # 게이트 *전* file→local 병합(C-R2). ⚠️ baseline 은 비동기(C-R9 비결속)
        config = ConfigPort.read(adk)  # null 가능
        if config != null: BootStatePort.mergeFromFile(config)  # readNaiaConfig→saveConfig(merged)
  4. decision = BootDecision(adk.present(*첫 평가 snapshot*), BootStatePort.isOnboardingComplete())
        # ⚠️ baseline: adk.present=첫 렌더 snapshot(2b 의 복구 반영 X), onboarding=병합 전 localStorage. 이식=결정적(타이밍 비결속, C-R9)
  5. if decision == SetupRequired:                              # setup 분기 = §C (게이트만)
        AdkPathPort.detectRoot()       # defaultPath 제시(자동완료 X, 사용자 load/new 확정 — login disabled)
        return                         # 게이트 종료 (panel list 는 수행됨; initAuth 는 §D 와 무관히 진행)
     # decision == OnboardingOverlay → main shell 유지+우측 OnboardingWizard / == Main → 진행
```

### D. initAuth() — **게이트와 독립한 mount effect** (C-R6 MED)

baseline `initAuth()` 는 `showAdkSetup` 게이트와 무관하게 mount 시 항상 시작 — SetupRequired(path 부재)여도 잔존 `naia-config` 있으면 발신될 수 있음. ⚠️ **고정 순서**(C-R2):

```
initAuth():                            # boot() 게이트와 병렬(순차 종속 X)
  config = BootStatePort.loadLocalConfigWithSecrets()   # ⚠️ secrets 경로(initAuth 전용, C-R12). localStorage 잔존 포함, null 가능
  if naiaKey:        send(AuthUpdate)                     # ① App.tsx 539 (naiaKey 조건 → 무키 no-op)
  if config != null: send(NotifyConfig)                  # ② App.tsx 550 (config 조건, C-R12)
  if config != null: send(CredsUpdate)                   # ③ App.tsx 566 (config 조건)
  # send = StartupMessagePort.store(msg) + send(msg)
  # 방어 재발신(C-R3): 일반부팅서 auth_update 한 번 더(398·535) — naiaKey 조건(무키 no-op, deferred)
# ※ ConfigPort.write(forAgent) = 별도 debounce effect (initAuth 와 순서 결속 X) — C-R1 MED
```

### B. WorkspaceCenterPanel 마운트/onActivate — **boot 공통 아님** (C-R1 MED)

```
onWorkspacePanelMount(userRoot):       # ⚠️ 활성화와 분리된 마운트 effect (C-R8). onboarding 중엔 content-panel 미렌더 → 미발동
  result = WorkspacePort.setRoot(userRoot)
       on Err → contain+fallback: WORKSPACE_ROOT 사용 + BootStatePort.clearWorkspaceRoot() + workspaceReady (block 아님)
       on success → WorkspacePort.startWatch()   # ⚠️ 마운트서도 즉시 호출 (C-R9; 미활성 일반부팅도 watch 시작)
onWorkspacePanelActivate():
  WorkspacePort.startWatch()            # ⚠️ 인자 없음 — 활성화서 재호출 (C-R8)
onWorkspacePanelDeactivate():
  WorkspacePort.stopWatch()             # workspace_stop_watch (C-R2 MED)
```

### C. setup 완료 분기 (path 부재 → 사용자 new/load/use-existing/recreate 확정)

```
onSetupConfirm(mode, path):
  if mode in {new, recreate}:                   # ⚠️ 디렉터리 준비가 *먼저* (C-R4)
      AdkSetupPort.inspectAdkDir(path) → 조건부 cloneAdk/deleteAdk
  AdkSetupPort.initSettings(path)               # init_naia_settings(path) — mode 무관(C-R10)
  AdkSetupPort.copyBundledAssets(path)          # copy_bundled_assets(path) = asset:// scope 확장
  # ⚠️ local config = 기존 보존 X (merge 금지, C-R7) — 모드별 reset/replace + 완료조건(C-R5·C-R6):
  switch mode:
    load:         cfg = ConfigPort.read(path)
                  BootStatePort.replaceLocalConfig(cfg or base)   # 통째 교체
                  BootStatePort.setWorkspaceRoot(path)            # ⚠️ 선택 path 강제 보존(preserveWorkspaceRoot, C-R12; 파일 stale/empty root 덮어씀)
                  BootStatePort.markOnboardingComplete()          # 무조건 (handleLoadConfirm 234·285)
    use-existing: BootStatePort.resetLocalConfig()                # 비우고 최소 재작성(먼저)
                  cfg = ConfigPort.read(path)
                  if cfg != null:
                      BootStatePort.replaceLocalConfig(cfg); BootStatePort.setWorkspaceRoot(path)  # 선택 path 강제 보존(C-R12)
                      BootStatePort.markOnboardingComplete()
                  # cfg null → 미완료(overlay 재진입) (handleNewUseExisting 292)
    new/recreate: BootStatePort.resetLocalConfig()                # 비우고 최소 재작성
                  BootStatePort.setWorkspaceRoot(path)            # workspaceRoot 만 — markOnboardingComplete 안 함 → overlay (213·267, C-R6)
  AdkPathPort.set(path)                         # localStorage + path 캐시 → 이후 boot() 재평가
```
> 순서 근거(C-R4): new/recreate = inspect/clone/delete → init_naia_settings → copy_bundled_assets. **완료 판정 직후 App 이 `isOnboardingComplete()`로 오버레이 진입 결정**(App.tsx 735): use-existing 조건부 완료 누락(C-R5)·new/recreate 의 미완료(C-R6) 둘 다 즉시 관측 드리프트.

> control-plane = 운영 로직만. 어떤 LLM/기억/인지도 없음. (인지 슬라이스 = F2+ 별도.)

## 4. adapters/ (driven 구현 — Tauri invoke 래퍼, **스캐폴드 시 stub**)

| 어댑터 | 구현 포트 | 호출 Tauri 커맨드 |
|---|---|---|
| `TauriConfigAdapter` | ConfigPort | `read_naia_config` / `write_naia_config`(+`buildNaiaConfigEnv` 파생 env 병합) |
| `TauriBootStateAdapter` | BootStatePort | localStorage `loadConfig`(onboarding flag) |
| `TauriAdkPathAdapter` | AdkPathPort | localStorage + `write_naia_path_cache` / `workspace_detect_adk_root` |
| `TauriWorkspaceAdapter` | WorkspacePort | `workspace_set_root`(canonicalize/is_dir 백엔드) / `workspace_start_watch` / `workspace_stop_watch` — 패널 lifecycle 트리거 |
| `TauriStartupAdapter` | StartupMessagePort | `store_startup_message` / `send_to_agent_command` |
| `TauriCredentialAdapter` | CredentialStorePort【deferred】 | `write_agent_key`(OS keychain) |
| `TauriPanelAdapter` | PanelInventoryPort | `panel_list_installed` |
| `TauriAdkSetupAdapter` | AdkSetupPort | `init_naia_settings`(먼저) / `copy_bundled_assets` / `inspect_adk_dir` / `clone_naia_adk` / `delete_naia_adk` |

> 직렬화·invoke·canonicalize 같은 I/O 세부는 어댑터/백엔드에만. 교체 시 app/domain 무변경.
> **팬텀 제외**: `check_naia_settings` = 부팅경로 미호출 → 어댑터/포트 없음.

## 6. ⚠️ 미확정 — 코드 스캐폴드 전 결정 필요(루크)

> **범위 메모(deferred)**: `CredentialStorePort`·AuthUpdate(initAuth+방어 재발신) = **외부키(naiaKey/apiKey) 영역 = UC12-min 무키 부팅엔 no-op**. 계약엔 인벤토리 완전성 위해 등재하되, F0 무키 슬라이스의 계약 테스트 대상 아님(V1 외부 auth 단계). 커버리지 manifest deferred 와 정합.


- **툴체인**: new-naia-os = package.json/tsconfig 없는 순수 docs 스캐폴드. 코드 스캐폴드 = TS 툴체인(package.json·tsconfig·dependency-cruiser·vitest) 신설 = **F13 root-files(헌장) 수정 + 런타임 토대 결정**. AI 단독 금지 → 루크 승인 후.
- **언어/런타임**: 위 시그니처 = 언어중립. 실제 = (a) shell 측 TS(invoke 래퍼) 헥사고날, (b) Rust 측 분리, (c) 혼합 중 결정.
- 이 계약은 **언어 무관 안정** — 툴체인 정해지면 그대로 코드화.
