## 1. domain/ (순수 값객체·규칙, import 0)

| 값객체 | 규칙(불변식) | baseline 근거 |
|---|---|---|
| `WorkspaceRoot` | raw path → `canonicalize`(symlink/mount/대소문자 정규화) + `is_dir` 검증. 실패 시 `Invalid`. **canonical 형태만 외부 노출**(FR-F1.4). | `workspace_set_root`(dunce::canonicalize, is_dir) |
| `NaiaConfig` | 필드 분류 규칙: `secret`(키/크레덴셜)·`ui`(표시전용)·`agent`(에이전트 노출 가능). **`forAgent()` = secret+ui 제거** 후 agent-facing 투영 산출(순수 변환). | `write_naia_config`의 `stripForAgent()` |
| `AdkPath` | 존재/유효(localStorage 값) — `Present(path)` \| `Absent`. | `isAdkInitialized`/`getAdkPath` |
| `BootDecision` | 순수 결정: `(adkPathPresent, onboardingComplete)` → `SetupRequired`(path 부재) \| `OnboardingOverlay`(path 有·onboarding 미완 → main shell 유지+우측 content 대체) \| `Main`(둘 다). | App.tsx showAdkSetup/onboarding 게이트(codex R1·R3) |
| `StartupMessage` | 종류 `NotifyConfig`\|`CredsUpdate`\|`AuthUpdate`. **발신 조건 규칙**: NotifyConfig/CredsUpdate = `config 존재`(loadConfigWithSecrets non-null) 필요; AuthUpdate = 추가로 `naiaKey 존재` 필요. | App.tsx 부팅 effect(codex R10·R12) |

> domain 은 어떤 포트/어댑터/transport 도 모름. 위 규칙은 전부 입력→출력 순수 함수.

## 2. ports/ (인터페이스 = 계약, domain 만 의존)

전부 **driven 포트**(app 이 *사용*). 시그니처 = 언어중립.

```
# ports/protocol  — transport-neutral 의미 payload (직렬화/프레이밍 누출 금지)
StartupMessagePayload  = { kind: NotifyConfig|CredsUpdate|AuthUpdate, body: <semantic struct> }
PanelDescriptor        = { id, title, ... }   # listInstalled 결과 의미형

# ports/  — driven 인터페이스
ConfigPort:
    read(adkPath: AdkPath): NaiaConfig | null            # 없으면 null (config.ts 295)
    write(adkPath: AdkPath, agentView: NaiaConfig.forAgent): void
                                                          # 호출자가 domain forAgent() 결과를 넘김(strip=domain)
AdkPathPort:
    get(): AdkPath                                        # Present|Absent
    set(path): void                                       # localStorage + path 캐시(홈 디렉터리 .naia 하위) 둘 다
    detectRoot(): AdkPath | null                          # 자동 *제시*만(자동완료 X)
WorkspacePort:
    setRoot(root: WorkspaceRoot): Result<CanonicalRoot, Err>   # Err = 호출자가 contain+fallback
    startWatch(root: CanonicalRoot): void                 # set_root 후 + 패널 onActivate
StartupMessagePort:
    store(msg: StartupMessagePayload): void               # replay cache (재시작용)
    send(msg: StartupMessagePayload): void                # 라이브 전송
PanelInventoryPort:
    listInstalled(): PanelDescriptor[]                    # 실패 가능 → 호출자 non-fatal contain
AdkBootstrapPort:                                          # setup 완료 분기(path 부재)
    copyBundledAssets(): void                             # 현 구현=asset:// scope 확장(실제 복사 X)
    inspectAdkDir(path): AdkDirState                      # new 경로 선행
    cloneAdk(path): void / deleteAdk(path): void          # 조건부
```

> ⚠️ 불변식(STRUCTURE.md R2): `StartupMessagePayload`·`PanelDescriptor` 에 wire-framing(JSON 라인·길이 prefix·gRPC) 누출 금지 — 직렬화는 transport adapter 만. stdio→gRPC = adapter 교체로 끝.

## 3. app/control/ (포트 사용 = 부팅 오케스트레이션, 인지 0)

`ControlPlaneBoot` — 포트만 사용(adapter 직접 import 금지). baseline 순서를 조건부로 재현:

```
boot():
  1. adk = AdkPathPort.get()
  2. decision = BootDecision(adk.present, onboardingComplete())
  3. if decision == SetupRequired:
        AdkPathPort.detectRoot()         # defaultPath 제시(자동완료 X, 사용자 load/new 확정 — login disabled)
        # setup 완료 시: AdkBootstrapPort.copyBundledAssets() (+ new: inspectAdkDir → 조건부 clone/delete)
        return                           # 사용자 확정까지 대기
  4. config = ConfigPort.read(adk)       # null 가능
  5. if config != null:                  # ← 조건부(codex R12): config 없으면 4건 스킵
        ConfigPort.write(adk, config.forAgent())            # strip=domain
        for msg in [NotifyConfig, CredsUpdate (+AuthUpdate if naiaKey)]:
            StartupMessagePort.store(msg); StartupMessagePort.send(msg)
  6. root = WorkspacePort.setRoot(userRoot)
        on Err → contain+fallback: WORKSPACE_ROOT + stale config clear + 진행 (block 아님)
        # 단 이 경로는 main content(WorkspaceCenterPanel) 마운트 후; onboarding 중엔 미발동
  7. WorkspacePort.startWatch(root)
  8. try PanelInventoryPort.listInstalled() catch → non-fatal(진행)
```

> control-plane = 운영 로직만. 어떤 LLM/기억/인지도 없음. (인지 슬라이스 = F2+ 별도.)

## 4. adapters/ (driven 구현 — Tauri invoke 래퍼, **스캐폴드 시 stub**)

| 어댑터 | 구현 포트 | 호출 Tauri 커맨드 |
|---|---|---|
| `TauriConfigAdapter` | ConfigPort | `read_naia_config` / `write_naia_config`(+`buildNaiaConfigEnv` 파생 env 병합) |
| `TauriAdkPathAdapter` | AdkPathPort | localStorage + `write_naia_path_cache` / `workspace_detect_adk_root` |
| `TauriWorkspaceAdapter` | WorkspacePort | `workspace_set_root` / `workspace_start_watch` |
| `TauriStartupAdapter` | StartupMessagePort | `store_startup_message` / `send_to_agent_command` |
| `TauriPanelAdapter` | PanelInventoryPort | `panel_list_installed` |
| `TauriAdkBootstrapAdapter` | AdkBootstrapPort | `copy_bundled_assets` / `inspect_adk_dir` / `clone_naia_adk` / `delete_naia_adk` |

> 직렬화·invoke 세부는 어댑터에만. 교체 시 app/domain 무변경.
> **팬텀 제외**: `check_naia_settings` = 부팅경로 미호출 → 어댑터/포트 없음.
