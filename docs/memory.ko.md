# Naia 메모리 시스템 — 사용자 가이드

Naia는 대화 사이에 당신에 대한 것들을 기억합니다. 이 가이드는 무엇이 저장되는지, 어디에, 그리고 어떻게 관리하는지 설명합니다.

[English](memory.md)

---

## Naia는 무엇을 기억하나요?

Naia는 대화에서 자동으로 사실(fact)을 추출합니다 — 당신이 자신에 대해 말한 것들, 선호도, 결정 사항들.

**Naia가 학습하는 사실의 예:**
- "나는 프론트엔드 개발자야"
- "나는 항상 Figma를 디자인에 써"
- "다크 모드를 선호해"
- "내 프로젝트 스택은 Next.js + FastAPI야"

모든 메시지가 저장되는 건 아닙니다. Naia는 중요도 점수(새로움, 관련성, 감정적 무게)를 사용해 무엇을 보관할지 결정합니다.

---

## 어디에 저장되나요?

모든 메모리는 **내 컴퓨터에 로컬로** 저장됩니다:

```
~/.naia/memory/alpha-memory.json
```

**Nextain 서버로는 아무것도 전송되지 않습니다** — 대화도, 팩트도, 선호도도. 메모리 파일은 당신이 명시적으로 백업하지 않는 한 기기를 벗어나지 않습니다.

---

## 메모리 작동 방식

Naia의 메모리는 4개의 레이어로 구성됩니다:

| 레이어 | 역할 |
|--------|------|
| **에피소드** | 원시 대화 턴 (언제, 무슨 말을 했는지) |
| **팩트** | 에피소드에서 추출된 지식 (선호도, 결정 사항) |
| **반성** | 과거 실패에서 Naia가 배운 교훈 |
| **작업 기억** | 현재 세션의 활성 컨텍스트 |

**통합(consolidation) 사이클:**
1. 대화 중 중요한 메시지가 **에피소드**로 저장됩니다
2. 30분마다 (비활성 시 5분 후) Naia가 "하루를 돌아보며" 에피소드에서 **팩트**를 추출합니다
3. 새 대화를 시작하면 관련 팩트와 최근 에피소드가 컨텍스트로 로드됩니다

## 메모리 관리

### 팩트 목록 보기

Naia 앱에서 **설정 → 메모리**를 여세요. Naia가 추출한 모든 팩트 목록이 보입니다.

### 팩트 삭제

**설정 → 메모리**에서 삭제하고 싶은 팩트 옆의 삭제(🗑) 버튼을 클릭하세요.

### 메모리 전체 초기화

**설정 → 메모리**에서 **"전체 초기화"**를 사용하면 모든 것이 삭제됩니다. Naia가 새로 시작합니다.

파일을 직접 삭제할 수도 있습니다:

```bash
rm ~/.naia/memory/alpha-memory.json
```

---

## 백업 및 복원

### 수동 백업

메모리 파일을 안전한 위치에 복사하세요:

```bash
cp ~/.naia/memory/alpha-memory.json ~/Documents/naia-memory-backup-$(date +%Y%m%d).json
```

### 백업에서 복원

```bash
cp ~/Documents/naia-memory-backup-YYYYMMDD.json ~/.naia/memory/alpha-memory.json
```

복원 후 Naia 앱을 재시작하세요.

### 새 컴퓨터로 이전

`~/.naia/memory/alpha-memory.json`을 새 컴퓨터의 같은 경로에 복사하세요. Naia가 전체 메모리를 그대로 이어받습니다.

---

## 개인정보 보호

- 메모리는 로컬 파일시스템에 일반 JSON으로 저장됩니다
- 기본적으로 암호화가 적용되지 않습니다 (파일시스템 권한으로 보호됩니다)
- Nextain은 메모리 데이터에 절대 접근할 수 없습니다
- Naia를 제거해도 메모리 파일은 `~/.naia/memory/`에 남습니다 — 필요하면 직접 삭제하세요

## 자주 묻는 질문

**Q: Naia가 내가 한 말을 기억하지 못해요.**

모든 메시지가 저장되는 건 아닙니다 — 중요도 임계값을 넘는 것만 저장됩니다. 짧거나 캐주얼하거나 새롭지 않은 메시지는 저장되지 않을 수 있습니다. 중요한 사실은 명시적으로 말해주세요: "나는 TypeScript를 써, 기억해둬."

**Q: Naia가 잘못 기억하고 있어요 — 어떻게 수정하나요?**

설정 → 메모리에서 잘못된 팩트를 찾아 삭제하세요. 그런 다음 올바른 정보를 Naia에게 말하면 다음 통합 사이클에 저장됩니다.

**Q: 원시 메모리 파일을 볼 수 있나요?**

네 — `~/.naia/memory/alpha-memory.json`에 일반 JSON으로 저장되어 있습니다. 어떤 텍스트 편집기로도 읽거나 수정할 수 있습니다.

**Q: 메모리가 성능에 영향을 미치나요?**

일반적인 규모(수백 개의 팩트)에서는 세션 시작에 50ms 미만이 추가됩니다. 통합 사이클은 백그라운드에서 실행되어 대화 지연에 영향을 주지 않습니다.
