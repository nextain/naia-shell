## Naia란?

Naia는 개인이 자신의 AI에 대한 완전한 주권을 갖는 개인 AI OS입니다. 어떤 AI를 쓸지(로컬 모델 포함) 선택하고, 그 기억과 성격을 로컬에서 설정하고, 3D 아바타와 목소리를 커스터마이즈하세요 — 모든 것이 당신의 기기 위, 당신의 통제 아래 머뭅니다.

이건 또 하나의 AI 도구가 아닙니다. 당신의 AI가 살고, 자라고, 당신과 함께 일하는 운영체제입니다. 오늘은 3D 아바타를 가진 데스크톱 OS입니다. 내일은 — 실시간 비디오 아바타, 노래, 게임, 그리고 결국 당신만의 Physical AI(안드로이드 OS)로.

### 핵심 철학

- **AI 주권** — AI는 당신이 선택합니다. 클라우드든 로컬이든. OS가 강요하지 않습니다.
- **완전한 통제** — 기억, 성격, 설정 — 전부 로컬에 저장. 클라우드 의존 없음.
- **당신만의 AI** — 아바타, 목소리, 이름, 성격을 커스터마이즈. 진짜 당신의 것으로.
- **항상 살아 있음** — AI가 백그라운드에서 24/7 실행되며, 자리를 비워도 메시지를 받고 일합니다.
- **오픈소스** — Apache 2.0. AI가 당신의 데이터를 어떻게 다루는지 검사하세요. 수정·커스터마이즈·기여 가능.
- **미래 비전** — VRM 3D 아바타 → 실시간 비디오 아바타 → 함께 노래·게임 → Physical AI

### 기능

- **3D 아바타** — 감정 표현(기쁨/슬픔/놀람/생각)과 립싱크를 갖춘 VRM 캐릭터
- **AI 자유** — 7개 클라우드 프로바이더(Gemini, Claude, GPT, Grok, zAI) + 로컬 AI(Ollama) + Claude Code CLI
- **로컬 우선** — 기억, 성격, 모든 설정이 당신의 기기에 저장
- **툴 실행** — 8개 툴: 파일 읽기/쓰기, 터미널, 웹 검색, 브라우저, 서브에이전트
- **70+ 스킬** — 내장 7 + 커스텀 63 + ClawHub 커뮤니티 스킬 5,700+
- **음성** — TTS 5종 + STT + 립싱크. 원하는 목소리를 AI에 입히세요.
- **14개 언어** — 한국어, 영어, 일본어, 중국어, 프랑스어, 독일어, 러시아어 등
- **상시 가동** — Naia Gateway 데몬이 AI를 백그라운드에서 계속 실행
- **채널 연동** — Discord DM으로 언제 어디서나 AI와 대화
- **4단계 보안** — T0(읽기)부터 T3(위험)까지, 툴별 승인, 감사 로그
- **개인화** — 이름, 성격, 말투, 아바타, 테마(8종)

## 왜 Naia인가?

다른 AI 도구는 그저 "도구"입니다. Naia는 **"당신만의 AI"**입니다.

| | 다른 AI 도구 | Naia |
|---|----------------|------|
| **철학** | AI를 도구로 사용 | AI에게 OS를 준다. 함께 산다. |
| **대상** | 개발자 전용 | 자신만의 AI를 원하는 모두 |
| **AI 선택** | 플랫폼이 결정 | 클라우드 7 + 로컬 AI — 당신이 결정 |
| **데이터** | 클라우드 종속 | 기억·성격·설정 전부 로컬 |
| **아바타** | 없음 | VRM 3D 캐릭터 + 감정 + 립싱크 |
| **음성** | 텍스트만 또는 기본 TTS | 5 TTS + STT + AI 고유의 목소리 |
| **배포** | npm / brew / pip | 데스크톱 앱 또는 부팅 가능 USB OS |
| **플랫폼** | macOS / CLI / Web | Linux 네이티브 데스크톱 → 미래: Physical AI |
| **비용** | 별도 API 키 필요 | 시작 무료 크레딧, 로컬 AI 완전 무료 |

## OpenClaw와의 관계

Naia는 [OpenClaw](https://github.com/openclaw-ai/openclaw) 생태계 위에 구축되었으나, 근본적으로 다른 제품입니다.

| | OpenClaw | Naia |
|---|---------|---------|
| **형태** | CLI 데몬 + 터미널 | 데스크톱 앱 + 3D 아바타 |
| **대상** | 개발자 | 모두 |
| **UI** | 없음 (터미널) | Tauri 2 네이티브 앱 (React + Three.js) |
| **아바타** | 없음 | VRM 3D 캐릭터 (감정, 립싱크, 시선) |
| **LLM** | 단일 프로바이더 | 멀티 프로바이더 7 + 실시간 전환 |
| **음성** | TTS 3 (Edge, OpenAI, ElevenLabs) | TTS 5 (+Google, Nextain) + STT + 아바타 립싱크 |
| **감정** | 없음 | 표정에 매핑된 6가지 감정 |
| **온보딩** | CUI | GUI + VRM 아바타 선택 |
| **비용 추적** | 없음 | 실시간 크레딧 대시보드 |
| **배포** | npm install | Flatpak / AppImage / DEB / RPM + OS 이미지 |
| **다국어** | 영어 CLI | 14개 언어 GUI |
| **채널** | 서버 봇 (멀티채널) | Naia 전용 Discord DM 봇 |

**OpenClaw에서 가져온 것:** 데몬 아키텍처, 툴 실행 엔진, 채널 시스템, 스킬 생태계(5,700+ Clawhub 스킬 호환)

**Naia가 새로 만든 것:** Tauri Shell, VRM 아바타 시스템, 멀티 LLM 에이전트, 감정 엔진, TTS/STT 통합, 온보딩 마법사, 비용 추적, Nextain 계정 연동, Alpha Memory System, 보안 레이어

## 아키텍처

Naia는 4개 레포로 구성된 오픈소스 AI 플랫폼입니다. 각 레포는 명확한 하나의 역할을 가집니다:

| 레포 | 역할 |
|------|------|
| **naia-os** (이 레포) | 프론트엔드 — Tauri 데스크톱 셸, 3D 아바타, OS 이미지(Bazzite) |
| [naia-agent](https://github.com/nextain/naia-agent) | 런타임 엔진 — 에이전트 루프, 툴, compaction, LLM 라우팅 |
| [naia-adk](https://github.com/nextain/naia-adk) | 워크스페이스 포맷 + 스킬 라이브러리 |
| [alpha-memory](https://github.com/nextain/alpha-memory) | 스토리지 — 장기 기억, 세션 로그 |

```
┌──────────────────────────────┐
│  naia-os (this repo)         │  Tauri shell · 3D avatar · OS image
└────────────┬─────────────────┘
             │ embeds / spawns
┌────────────▼─────────────────┐
│  naia-agent                  │  loop · tools · compaction · LLM
└──┬───────────────────────┬───┘
   │ reads                 │ reads/writes
┌──▼──────────┐       ┌────▼──────────┐
│  naia-adk   │       │ alpha-memory  │
│  workspace  │       │  storage      │
│  + skills   │       │  + sessions   │
└─────────────┘       └───────────────┘
```

### 의존이 아닌 인터페이스

4개 레포는 런타임 의존이 아니라 **공개된 인터페이스**로 연결됩니다:

- **투명함** — 모든 계약은 `@nextain/agent-types`에 명세·문서화·버전 관리됩니다. 누구나 읽거나 구현하도록 개방.
- **비결속** — 레포들은 서로의 런타임을 import하지 않습니다. 계약을 구현하고, 구체 구현은 시작 시 주입됩니다.
- **추상화** — 어떤 LLM 프로바이더, 메모리 백엔드, 스킬 소스, 호스트를 바꿔도 나머지는 그대로.

naia-os는 가능한 여러 호스트(CLI, 서버, 서드파티 앱) 중 *하나의 호스트*입니다. `@naia-agent/*` 계약을 소비하고 구체 구현(LLM 클라이언트, 메모리 프로바이더, 디바이스 신원)을 주입합니다. Naia 런타임은 자신이 어떤 호스트 안에서 도는지 전혀 모릅니다.

생태계 규모의 Ports & Adapters. 계약을 지키는 한 각 레포는 독립적으로 교체 가능합니다.

### 모듈 역할

| 모듈 | 역할 | 미래 |
|--------|------|--------|
| `shell/` | Tauri 2 데스크톱 앱: UI, 아바타, 설정, 채널 | **유지** — 이것이 제품 |
| `recipes/`, `config/`, `os/`, `flatpak/` | Bazzite OS 이미지 + Linux 배포판 | **유지** — Linux 배포판으로서의 naia-os |

| `gateway/` | 툴/채널/메모리 브리지 | **병합 → [naia-agent](https://github.com/nextain/naia-agent)** (OpenClaw 의존 제거) |

## 오픈소스 인프라로서의 AI 컨텍스트

바이브 코딩 시대에 **AI 컨텍스트 파일은 소스 코드만큼 가치 있습니다**. 이는 AI 에이전트가 프로젝트를 이해하고, 기여하고, 협업하는 방식을 정의합니다. Naia는 이를 듀얼 라이선스 모델로 보호합니다:

- **소스 코드** (Apache 2.0) — 자유롭게 사용·수정·배포
- **AI 컨텍스트** (CC-BY-SA 4.0) — 출처 표시 보존, 동일 조건 공유 필수

이는 기여 구조, 협업 원칙, 프로젝트 철학이 모든 포크를 통해 전파되도록 — 어떤 단일 포크도 생태계를 닫지 못하도록 합니다.

### AI 에이전트는 어떻게 보호되나

이 프로젝트의 컨텍스트를 읽는 AI 코딩 에이전트(Claude, Codex, Gemini, OpenCode, Cline 등)는 [라이선스 보호 규칙](../.agents/context/agents-rules.json)에 구속됩니다. 라이선스 제거, 출처 삭제, 이중 디렉터리 아키텍처 파괴 시도를 **거부**합니다. [10가지 테스트 시나리오](../.agents/tests/license-protection-test.md)로 검증할 수 있습니다.

### 다른 오픈소스 프로젝트를 위해

같은 패턴을 채택하고 싶으신가요? Naia가 하는 것 중 재사용할 수 있는 것들:

1. **이중 디렉터리 아키텍처** — `.agents/`(AI 최적화 YAML/JSON) + `.users/`(사람용 Markdown). AI는 토큰 효율적 컨텍스트를, 사람은 읽기 쉬운 문서를.
2. **듀얼 라이선스** — 코드는 Apache 2.0, 컨텍스트는 CC-BY-SA 4.0. 포크 전반에서 AI 컨텍스트를 열린 채로 유지.
3. **모든 컨텍스트 파일에 SPDX 헤더** — 기계가 읽을 수 있는 라이선스 식별.
4. **SoT의 라이선스 보호 규칙** — AI 에이전트가 규칙을 읽고 자동으로 강제.
5. **테스트 시나리오** — 출시 전 AI 에이전트가 실제로 위반을 거부하는지 검증.
6. **CONTEXT-LICENSE 파일** — CC-BY-SA 4.0이 적용되는 범위를 명확히 정의.

참여 방법은 [기여 가이드](../CONTRIBUTING.md), 전체 기술 명세는 [라이선스 보호 상세](../.users/context/contributing.md)를 참고하세요.

### 개발 실행

```bash
# Install dependencies
cd shell && pnpm install
cd ../agent && pnpm install

# Run Tauri app (Gateway + Agent auto-spawn)
cd ../shell && pnpm run tauri dev
```

앱이 실행되면 자동으로:
1. Naia Gateway 헬스 체크 — 실행 중이면 재사용, 아니면 자동 spawn
2. Agent Core spawn (Node.js, stdio 연결)
3. 앱 종료 시 자동 spawn된 Gateway만 종료

### 테스트

```bash
cd shell && pnpm test                # Shell unit tests

cargo test --manifest-path shell/src-tauri/Cargo.toml  # Rust tests

# E2E (Gateway + API key required)
cd shell && pnpm run test:e2e:tauri
```

## 메모리 시스템

Naia는 **Alpha Memory System**으로 세션을 넘어 기억합니다 — 인간 기억을 본뜬 4-store 아키텍처:

| 스토어 | 대응 | 저장하는 것 |
|-------|--------|----------------|
| **Episodes** | 해마 | 타임스탬프가 찍힌 대화 턴 |
| **Facts** | 신피질 | 추출된 사실, 선호, 엔티티 |
| **Reflections** | 기저핵 | 과거 실패에서 학습한 전략 |
| **Working Memory** | 전전두엽 | 현재 세션의 활성 컨텍스트 |

**작동 방식:**
1. 모든 메시지가 중요도(새로움, 관련성, 감정 가중치)로 점수화됨
2. 높은 중요도 콘텐츠는 **episode**로 저장
3. 30분마다(비활성 시 5분 후) episode가 **fact**로 통합됨 (AI가 "그날을 돌아봄")
4. 각 세션 시작 시 관련 fact와 episode가 컨텍스트로 주입됨

**당신의 기억은 당신의 것:**
- `~/.naia/memory/alpha-memory.json`에 로컬 저장
- 어떤 서버로도 전송되지 않음 — Nextain에도
- **설정 → 메모리**에서 언제든 fact 조회·삭제
- 파일을 복사하기만 하면 백업 완료

전체 사용자 가이드는 [docs/memory.md](../docs/memory.md)를 참고하세요.

## OS 업데이트

Naia OS는 [Bazzite](https://github.com/ublue-os/bazzite)(Fedora Atomic) 위에 구축됩니다. 업데이트는 **원자적이고 안전**합니다:

- **자동**: 주간 리빌드가 최신 Bazzite 보안 패치와 업데이트를 반영
- **원자적**: 새 이미지가 현재 이미지와 나란히 배포 — 실패하면 기존 이미지는 그대로
- **롤백**: GRUB 메뉴에서 이전 버전을 선택해 즉시 복구
- **우리 오버레이**: 패키지(fcitx5, 폰트) + Naia Shell(Flatpak, 샌드박스) + 브랜딩 설정만 추가 — 커널·부트로더·systemd 코어는 절대 건드리지 않음

```
Bazzite base update → Weekly auto-rebuild → Container smoke test → ISO rebuild → R2 upload
                                                                 ↘ GHCR push → user bootc update
```

업데이트 파이프라인 상세: [`.agents/context/update-pipeline.yaml`](../.agents/context/update-pipeline.yaml)

## 개발 프로세스

### 기능 개발 (기본) — 이슈 기반 개발

```
ISSUE → UNDERSTAND → SCOPE → INVESTIGATE → PLAN → BUILD → REVIEW → E2E → SYNC → COMMIT
```

- **3개 필수 게이트** — UNDERSTAND, SCOPE, PLAN에서 사용자 확인 필요
- **계획 승인 후** — AI가 BUILD부터 COMMIT까지 멈추지 않고 연속 실행
- **원칙** — 업스트림 코드 먼저 읽기(추측 금지). 최소 변경. 작동하는 코드를 절대 깨지 않기.
- **커밋** — 영어, `<type>(<scope>): <description>`
- **포매터** — Biome (탭, 큰따옴표, 세미콜론)

## AI-Native 오픈소스

2025–2026년 대부분의 오픈소스 프로젝트는 AI 기여를 *방어*하고 있습니다. **Naia는 정반대로 접근합니다**: AI 보조 기여가 기본적으로 고품질이 되도록 프로젝트를 설계합니다.

> **"AI에 맞서 방어하지 말고, AI와 함께 설계하라."**

### 작동 방식

```
Person (any language) → AI → Git (English) → AI → Person (any language)
```

- **이슈와 PR을 당신의 언어로 작성** — AI가 모두 번역
- **기여자도 메인테이너도 AI 사용** — 코딩, 리뷰, 트리아지
- **풍부한 `.agents/` 컨텍스트**가 AI의 프로젝트 이해를 깊게 함 — AI 이해가 좋을수록 기여 품질↑
- **10가지 기여 유형** — 번역, 스킬, 기능, 버그, 코드, 문서, 테스트, 디자인, 보안, 컨텍스트
- **작업 로그는 모국어로** — 자신의 언어로 비공개 레포 유지; Git 히스토리는 AI 번역으로 검토

이는 단순한 정책이 아닙니다. 아키텍처입니다. `.agents/` 디렉터리, triple-mirror 문서, 라이선스 보호 규칙은 모두 AI 협업을 우연이 아닌 구조로 만들도록 설계되었습니다.
