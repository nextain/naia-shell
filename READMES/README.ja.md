## Naiaとは?

Naiaは、個人が自分のAIに対する完全な主権を持つ個人向けAI OSです。どのAIを使うか(ローカルモデルを含む)を選び、その記憶と性格をローカルで設定し、3Dアバターと声をカスタマイズしてください — すべてがあなたの端末上、あなたの管理下に留まります。

これはもう一つのAIツールではありません。あなたのAIが暮らし、育ち、あなたと共に働くオペレーティングシステムです。今日は3Dアバターを備えたデスクトップOSです。明日は — リアルタイムビデオアバター、歌、ゲーム、そしてやがてあなただけのPhysical AI(アンドロイドOS)へ。

### 核心となる哲学

- **AI主権** — AIはあなたが選びます。クラウドでもローカルでも。OSが強制しません。
- **完全な管理** — 記憶、性格、設定 — すべてローカルに保存。クラウド依存なし。
- **あなただけのAI** — アバター、声、名前、性格をカスタマイズ。本当にあなたのものに。
- **常に生きている** — AIがバックグラウンドで24/7稼働し、席を外してもメッセージを受け取り働きます。
- **オープンソース** — Apache 2.0。AIがあなたのデータをどう扱うか検査できます。修正・カスタマイズ・貢献が可能。
- **未来のビジョン** — VRM 3Dアバター → リアルタイムビデオアバター → 共に歌・ゲーム → Physical AI

### 機能

- **3Dアバター** — 感情表現(喜び/悲しみ/驚き/思考)とリップシンクを備えたVRMキャラクター
- **AIの自由** — 7つのクラウドプロバイダー(Gemini, Claude, GPT, Grok, zAI) + ローカルAI(Ollama) + Claude Code CLI
- **ローカルファースト** — 記憶、性格、すべての設定があなたの端末に保存
- **ツール実行** — 8つのツール: ファイル読み書き、ターミナル、Web検索、ブラウザ、サブエージェント
- **70以上のスキル** — 内蔵7 + カスタム63 + ClawHubコミュニティスキル5,700以上
- **音声** — TTS 5種 + STT + リップシンク。好きな声をAIにまとわせてください。
- **14言語** — 韓国語、英語、日本語、中国語、フランス語、ドイツ語、ロシア語など
- **常時稼働** — Naia GatewayデーモンがAIをバックグラウンドで稼働し続けます
- **チャンネル連携** — Discord DMでいつでもどこでもAIと対話
- **4段階セキュリティ** — T0(読み取り)からT3(危険)まで、ツール別承認、監査ログ
- **パーソナライズ** — 名前、性格、口調、アバター、テーマ(8種)

## なぜNaiaなのか?

他のAIツールは単なる「ツール」です。Naiaは **「あなただけのAI」** です。

| | 他のAIツール | Naia |
|---|----------------|------|
| **哲学** | AIをツールとして使う | AIにOSを与える。共に暮らす。 |
| **対象** | 開発者専用 | 自分だけのAIを望むすべての人 |
| **AI選択** | プラットフォームが決定 | クラウド7 + ローカルAI — あなたが決定 |
| **データ** | クラウド依存 | 記憶・性格・設定すべてローカル |
| **アバター** | なし | VRM 3Dキャラクター + 感情 + リップシンク |
| **音声** | テキストのみ または基本TTS | 5 TTS + STT + AI固有の声 |
| **配布** | npm / brew / pip | デスクトップアプリ または起動可能なUSB OS |
| **プラットフォーム** | macOS / CLI / Web | Linuxネイティブデスクトップ → 未来: Physical AI |
| **コスト** | 別途APIキーが必要 | 開始無料クレジット、ローカルAI完全無料 |

## アーキテクチャ

Naiaは4つのレポで構成されるオープンソースAIプラットフォームです。各レポは明確な一つの役割を持ちます:

| レポ | 役割 |
|------|------|
| **naia-os** (このレポ) | フロントエンド — Tauriデスクトップシェル、3Dアバター、OSイメージ(Bazzite) |
| [naia-agent](https://github.com/nextain/naia-agent) | ランタイムエンジン — エージェントループ、ツール、compaction、LLMルーティング |
| [naia-adk](https://github.com/nextain/naia-adk) | ワークスペースフォーマット + スキルライブラリ |
| [alpha-memory](https://github.com/nextain/alpha-memory) | ストレージ — 長期記憶、セッションログ |

```
┌──────────────────────────────┐
│  naia-os (this repo)         │  Tauri shell · 3D avatar · OS image
└────────────┬─────────────────┘
             │ embeds / spawns
┌────────────▼─────────────────┐
│  naia-agent                  │  loop · tools · compaction · LLM
└──┬───────────────────────┬───┘
   │ reads                 │ reads/writes
┌──▼──────────┐       ┌────▼──────────┐
│  naia-adk   │       │ alpha-memory  │
│  workspace  │       │  storage      │
│  + skills   │       │  + sessions   │
└─────────────┘       └───────────────┘
```

### 依存ではなくインターフェース

4つのレポはランタイム依存ではなく **公開されたインターフェース** で接続されます:

- **透明性** — すべての契約は `@nextain/agent-types` に明記・文書化・バージョン管理されます。誰でも読み、実装できるよう開放。
- **非結合** — レポ同士は互いのランタイムをimportしません。契約を実装し、具体実装は起動時に注入されます。
- **抽象化** — どのLLMプロバイダー、メモリバックエンド、スキルソース、ホストを変えても残りはそのまま。

naia-osは可能な複数のホスト(CLI、サーバー、サードパーティアプリ)のうちの *一つのホスト* です。`@naia-agent/*` 契約を消費し、具体実装(LLMクライアント、メモリプロバイダー、デバイス身元)を注入します。Naiaランタイムは自分がどのホストの中で動いているか一切知りません。

エコシステム規模のPorts & Adapters。契約を守る限り各レポは独立して交換可能です。

## オープンソースインフラとしてのAIコンテキスト

バイブコーディング時代において **AIコンテキストファイルはソースコードと同じくらい価値があります**。これはAIエージェントがプロジェクトを理解し、貢献し、協業する方法を定義します。Naiaはこれをデュアルライセンスモデルで保護します:

- **ソースコード** (Apache 2.0) — 自由に使用・修正・配布
- **AIコンテキスト** (CC-BY-SA 4.0) — 出典表示を保存、同一条件での共有が必須

これは貢献構造、協業原則、プロジェクト哲学があらゆるフォークを通じて伝播するように — どの単一のフォークもエコシステムを閉じられないようにします。

### AIエージェントはどう保護されるのか

このプロジェクトのコンテキストを読むAIコーディングエージェント(Claude, Codex, Gemini, OpenCode, Clineなど)は [ライセンス保護ルール](../.agents/context/agents-rules.json) に拘束されます。ライセンス除去、出典削除、二重ディレクトリアーキテクチャの破壊の試みを **拒否** します。[10種類のテストシナリオ](../.agents/tests/license-protection-test.md) で検証できます。

### 他のオープンソースプロジェクトのために

同じパターンを採用したいですか? Naiaが行っていることのうち再利用できるもの:

1. **二重ディレクトリアーキテクチャ** — `.agents/`(AI最適化YAML/JSON) + `.users/`(人間用Markdown)。AIにはトークン効率の良いコンテキストを、人間には読みやすいドキュメントを。
2. **デュアルライセンス** — コードはApache 2.0、コンテキストはCC-BY-SA 4.0。フォーク全体でAIコンテキストを開かれたまま維持。
3. **すべてのコンテキストファイルにSPDXヘッダー** — 機械が読めるライセンス識別。
4. **SoTのライセンス保護ルール** — AIエージェントがルールを読み自動的に強制。
5. **テストシナリオ** — リリース前にAIエージェントが実際に違反を拒否するか検証。
6. **CONTEXT-LICENSEファイル** — CC-BY-SA 4.0が適用される範囲を明確に定義。

参加方法は [貢献ガイド](../CONTRIBUTING.md)、全技術仕様は [ライセンス保護の詳細](../.users/context/contributing.md) を参考にしてください。

## クイックスタート

### 前提条件

- Linux (Bazzite, Ubuntu, Fedoraなど)
- Node.js 22+, pnpm 9+
- Rust stable (Tauriビルド用)
- システムパッケージ (Fedora): `webkit2gtk4.1-devel libappindicator-gtk3-devel librsvg2-devel`
- cmake (whisper.cppビルド用)

### 開発実行

```bash
# Install dependencies
cd shell && pnpm install
cd ../agent && pnpm install

# Run Tauri app (Gateway + Agent auto-spawn)
cd ../shell && pnpm run tauri dev
```

アプリが実行されると自動的に:
1. Naia Gatewayヘルスチェック — 実行中なら再利用、そうでなければ自動spawn
2. Agent Core spawn (Node.js, stdio接続)
3. アプリ終了時に自動spawnされたGatewayのみ終了

### テスト

```bash
cd shell && pnpm test                # Shell unit tests
cd agent && pnpm test                # Agent unit tests
cd agent && pnpm exec tsc --noEmit   # Type check
cargo test --manifest-path shell/src-tauri/Cargo.toml  # Rust tests

# E2E (Gateway + API key required)
cd shell && pnpm run test:e2e:tauri
```

### Flatpakビルド

```bash
flatpak install --user flathub org.freedesktop.Platform//24.08 org.freedesktop.Sdk//24.08
flatpak-builder --user --install --force-clean build-dir flatpak/io.nextain.naia.yml
flatpak run io.nextain.naia
```

## セキュリティモデル

Naiaは **多層防御(Defense in Depth)** セキュリティモデルを適用します:

| レイヤー | 保護 |
|-------|-----------|
| OS | Bazzite不変rootfs + SELinux |
| Gateway | OpenClawデバイス認証 + トークンスコープ |
| Agent | 4段階権限 (T0~T3) + ツール別ブロック |
| Shell | ユーザー承認モーダル + ツールON/OFFトグル |
| Audit | SQLite監査ログ (すべてのツール実行を記録) |

## メモリシステム

Naiaは **Alpha Memory System** でセッションを越えて記憶します — 人間の記憶を模した4-storeアーキテクチャ:

| ストア | 対応 | 保存するもの |
|-------|--------|----------------|
| **Episodes** | 海馬 | タイムスタンプが押された会話ターン |
| **Facts** | 新皮質 | 抽出された事実、選好、エンティティ |
| **Reflections** | 大脳基底核 | 過去の失敗から学んだ戦略 |
| **Working Memory** | 前頭前野 | 現在のセッションのアクティブコンテキスト |

**仕組み:**
1. すべてのメッセージが重要度(新規性、関連性、感情的重み)でスコア化される
2. 高い重要度のコンテンツは **episode** として保存
3. 30分ごと(非活動時は5分後)にepisodeが **fact** へ統合される (AIが「その日を振り返る」)
4. 各セッション開始時に関連するfactとepisodeがコンテキストとして注入される

**あなたの記憶はあなたのもの:**
- `~/.naia/memory/alpha-memory.json` にローカル保存
- どのサーバーにも送信されません — Nextainにも
- **設定 → メモリ** でいつでもfactを照会・削除
- ファイルをコピーするだけでバックアップ完了

完全なユーザーガイドは [docs/memory.md](../docs/memory.md) を参考にしてください。

## OSアップデート

Naia OSは [Bazzite](https://github.com/ublue-os/bazzite)(Fedora Atomic) の上に構築されます。アップデートは **原子的で安全** です:

- **自動**: 週次リビルドが最新のBazziteセキュリティパッチとアップデートを反映
- **原子的**: 新しいイメージが現在のイメージと並んでデプロイ — 失敗すれば既存イメージはそのまま
- **ロールバック**: GRUBメニューから以前のバージョンを選んで即座に復旧
- **私たちのオーバーレイ**: パッケージ(fcitx5, フォント) + Naia Shell(Flatpak, サンドボックス) + ブランディング設定のみ追加 — カーネル・ブートローダー・systemdコアには絶対に手を触れない

```
Bazzite base update → Weekly auto-rebuild → Container smoke test → ISO rebuild → R2 upload
                                                                 ↘ GHCR push → user bootc update
```

アップデートパイプラインの詳細: [`.agents/context/update-pipeline.yaml`](../.agents/context/update-pipeline.yaml)

## 開発プロセス

### 機能開発 (基本) — イシューベースの開発

```
ISSUE → UNDERSTAND → SCOPE → INVESTIGATE → PLAN → BUILD → REVIEW → E2E → SYNC → COMMIT
```

- **3つの必須ゲート** — UNDERSTAND, SCOPE, PLANでユーザー確認が必要
- **計画承認後** — AIがBUILDからCOMMITまで止まらず連続実行
- **原則** — アップストリームコードを先に読む(推測禁止)。最小限の変更。動作するコードを絶対に壊さない。
- **コミット** — 英語、`<type>(<scope>): <description>`
- **フォーマッター** — Biome (タブ、ダブルクォート、セミコロン)

## AI-Nativeなオープンソース

2025–2026年、ほとんどのオープンソースプロジェクトはAIの貢献を *防御* しています。**Naiaは正反対のアプローチを取ります**: AI支援による貢献がデフォルトで高品質になるようプロジェクトを設計します。

> **「AIに対して防御するな、AIと共に設計せよ。」**

### 仕組み

```
Person (any language) → AI → Git (English) → AI → Person (any language)
```

- **イシューとPRをあなたの言語で作成** — AIがすべて翻訳
- **貢献者もメンテナーもAIを使用** — コーディング、レビュー、トリアージ
- **豊富な `.agents/` コンテキスト** がAIのプロジェクト理解を深める — AIの理解が良いほど貢献品質↑
- **10種類の貢献タイプ** — 翻訳、スキル、機能、バグ、コード、ドキュメント、テスト、デザイン、セキュリティ、コンテキスト
- **作業ログは母国語で** — 自分の言語で非公開レポを維持; GitヒストリーはAI翻訳でレビュー

これは単なるポリシーではありません。アーキテクチャです。`.agents/` ディレクトリ、triple-mirrorドキュメント、ライセンス保護ルールはすべて、AI協業を偶然ではなく構造にするよう設計されています。

完全なモデルを読む: [`open-source-operations.yaml`](../.agents/context/open-source-operations.yaml) | [Report (EN)](../docs/reports/20260307-ai-native-opensource-operations.md) | [Report (KO)](../docs/reports/20260307-ai-native-opensource-operations-ko.md)
