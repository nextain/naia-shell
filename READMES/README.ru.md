## Что такое Naia?

Naia — это персональная AI ОС, в которой человек обладает полным суверенитетом над своим ИИ. Выбирайте, какой ИИ использовать (включая локальные модели), настраивайте его память и характер локально, кастомизируйте 3D-аватар и голос — всё остаётся на вашем устройстве, под вашим контролем.

Это не очередной AI-инструмент. Это операционная система, в которой ваш ИИ живёт, растёт и работает вместе с вами. Сегодня это десктопная ОС с 3D-аватаром. Завтра — видеоаватар в реальном времени, пение, игры и, в конечном счёте, ваш собственный Physical AI (Android-ОС).

### Ключевая философия

- **Суверенитет ИИ** — ИИ выбираете вы. Облачный или локальный. ОС не навязывает.
- **Полный контроль** — память, характер, настройки — всё хранится локально. Без зависимости от облака.
- **Ваш собственный ИИ** — кастомизируйте аватар, голос, имя, характер. Сделайте его по-настоящему своим.
- **Всегда жив** — ИИ работает в фоне 24/7, получает сообщения и работает, даже когда вас нет.
- **Открытый код** — Apache 2.0. Проверяйте, как ИИ обращается с вашими данными. Можно изменять, кастомизировать и вносить вклад.
- **Видение будущего** — VRM 3D-аватар → видеоаватар в реальном времени → совместное пение и игры → Physical AI

### Возможности

- **3D-аватар** — VRM-персонаж с выражением эмоций (радость/грусть/удивление/задумчивость) и липсинком
- **Свобода ИИ** — 7 облачных провайдеров (Gemini, Claude, GPT, Grok, zAI) + локальный ИИ (Ollama) + Claude Code CLI
- **Локально в первую очередь** — память, характер и все настройки хранятся на вашем устройстве
- **Выполнение инструментов** — 8 инструментов: чтение/запись файлов, терминал, веб-поиск, браузер, субагент
- **70+ навыков** — 7 встроенных + 63 кастомных + 5 700+ навыков сообщества ClawHub
- **Голос** — 5 видов TTS + STT + липсинк. Придайте ИИ желаемый голос.
- **14 языков** — корейский, английский, японский, китайский, французский, немецкий, русский и др.
- **Постоянная работа** — демон Naia Gateway держит ИИ работающим в фоне
- **Интеграция каналов** — общайтесь с ИИ когда угодно и где угодно через Discord DM
- **4 уровня безопасности** — от T0 (чтение) до T3 (опасные), одобрение по каждому инструменту, журнал аудита
- **Персонализация** — имя, характер, манера речи, аватар, темы (8 видов)

## Почему Naia?

Другие AI-инструменты — это просто «инструменты». Naia — это **«ваш собственный ИИ»**.

| | Другие AI-инструменты | Naia |
|---|----------------|------|
| **Философия** | Использовать ИИ как инструмент | Дать ИИ ОС. Жить вместе. |
| **Аудитория** | Только для разработчиков | Все, кто хочет собственного ИИ |
| **Выбор ИИ** | Решает платформа | 7 облачных + локальный ИИ — решаете вы |
| **Данные** | Привязка к облаку | Память, характер, настройки — всё локально |
| **Аватар** | Нет | VRM 3D-персонаж + эмоции + липсинк |
| **Голос** | Только текст или базовый TTS | 5 TTS + STT + собственный голос ИИ |
| **Распространение** | npm / brew / pip | Десктопное приложение или загрузочная USB-ОС |
| **Платформа** | macOS / CLI / Web | Нативный десктоп Linux → будущее: Physical AI |
| **Стоимость** | Нужен отдельный API-ключ | Бесплатные кредиты на старте, локальный ИИ полностью бесплатно |

| | OpenClaw | Naia |
|---|---------|---------|
| **Форма** | CLI-демон + терминал | Десктопное приложение + 3D-аватар |
| **Аудитория** | Разработчики | Все |
| **UI** | Нет (терминал) | Нативное приложение Tauri 2 (React + Three.js) |
| **Аватар** | Нет | VRM 3D-персонаж (эмоции, липсинк, взгляд) |
| **LLM** | Один провайдер | Мульти-провайдер 7 + переключение в реальном времени |
| **Голос** | TTS 3 (Edge, OpenAI, ElevenLabs) | TTS 5 (+Google, Nextain) + STT + липсинк аватара |
| **Эмоции** | Нет | 6 эмоций, сопоставленных с мимикой |
| **Онбординг** | CUI | GUI + выбор VRM-аватара |
| **Отслеживание затрат** | Нет | Панель кредитов в реальном времени |
| **Распространение** | npm install | Flatpak / AppImage / DEB / RPM + образ ОС |
| **Многоязычность** | Англоязычный CLI | GUI на 14 языках |
| **Каналы** | Серверный бот (мультиканал) | Эксклюзивный для Naia Discord DM-бот |

**Что взято из OpenClaw:** архитектура демона, движок выполнения инструментов, система каналов, экосистема навыков (совместимость с 5 700+ навыками Clawhub)

**Что Naia создала заново:** Tauri Shell, система VRM-аватаров, мульти-LLM-агент, движок эмоций, интеграция TTS/STT, мастер онбординга, отслеживание затрат, интеграция аккаунта Nextain, Alpha Memory System, слой безопасности

## Архитектура

Naia — это AI-платформа с открытым кодом, состоящая из 4 репозиториев. У каждого репозитория одна чёткая роль:

| Репозиторий | Роль |
|------|------|
| **naia-os** (этот репозиторий) | Фронтенд — десктопная оболочка Tauri, 3D-аватар, образ ОС (Bazzite) |
| [naia-agent](https://github.com/nextain/naia-agent) | Движок рантайма — цикл агента, инструменты, compaction, маршрутизация LLM |
| [naia-adk](https://github.com/nextain/naia-adk) | Формат рабочего пространства + библиотека навыков |
| [alpha-memory](https://github.com/nextain/alpha-memory) | Хранилище — долговременная память, журналы сессий |

```
┌──────────────────────────────┐
│  naia-os (this repo)         │  Tauri shell · 3D avatar · OS image
└────────────┬─────────────────┘
             │ embeds / spawns
┌────────────▼─────────────────┐
│  naia-agent                  │  loop · tools · compaction · LLM
└──┬───────────────────────┬───┘
   │ reads                 │ reads/writes
┌──▼──────────┐       ┌────▼──────────┐
│  naia-adk   │       │ alpha-memory  │
│  workspace  │       │  storage      │
│  + skills   │       │  + sessions   │
└─────────────┘       └───────────────┘
```

### Интерфейс, а не зависимость

4 репозитория связаны не рантайм-зависимостью, а **публичными интерфейсами**:

- **Прозрачность** — все контракты специфицированы, документированы и версионированы в `@nextain/agent-types`. Открыты для чтения или реализации кем угодно.
- **Несвязанность** — репозитории не импортируют рантайм друг друга. Они реализуют контракт, а конкретная реализация внедряется при запуске.
- **Абстракция** — смените любой LLM-провайдер, бэкенд памяти, источник навыков или хост — остальное останется как есть.

naia-os — это *один из* возможных хостов (CLI, сервер, стороннее приложение). Он потребляет контракты `@naia-agent/*` и внедряет конкретные реализации (LLM-клиент, провайдер памяти, идентичность устройства). Рантайм Naia вообще не знает, внутри какого хоста он работает.

Ports & Adapters масштаба экосистемы. Пока контракт соблюдается, каждый репозиторий заменяем независимо.

## AI-контекст как инфраструктура с открытым кодом

В эпоху вайб-кодинга **файлы AI-контекста ценны не меньше исходного кода**. Они определяют, как AI-агенты понимают проект, вносят вклад и сотрудничают. Naia защищает это двойной лицензионной моделью:

- **Исходный код** (Apache 2.0) — свободное использование, изменение, распространение
- **AI-контекст** (CC-BY-SA 4.0) — обязательно сохранение указания авторства и распространение на тех же условиях

Это обеспечивает распространение структуры вкладов, принципов сотрудничества и философии проекта через все форки — так, чтобы ни один отдельный форк не мог закрыть экосистему.

### Как защищены AI-агенты

AI-агенты для написания кода (Claude, Codex, Gemini, OpenCode, Cline и др.), читающие контекст этого проекта, связаны [правилами защиты лицензии](../.agents/context/agents-rules.json). Они **отказываются** от попыток удалить лицензию, стереть указание авторства или разрушить архитектуру двойных директорий. Это можно проверить по [10 тестовым сценариям](../.agents/tests/license-protection-test.md).

### Для других проектов с открытым кодом

Хотите перенять тот же паттерн? Вот что у Naia можно переиспользовать:

1. **Архитектура двойных директорий** — `.agents/` (YAML/JSON, оптимизированный для ИИ) + `.users/` (Markdown для людей). ИИ получает токен-эффективный контекст, люди — удобную для чтения документацию.
2. **Двойная лицензия** — код под Apache 2.0, контекст под CC-BY-SA 4.0. AI-контекст остаётся открытым во всех форках.
3. **SPDX-заголовок в каждом файле контекста** — машиночитаемая идентификация лицензии.
4. **Правила защиты лицензии в SoT** — AI-агенты читают правила и автоматически их применяют.
5. **Тестовые сценарии** — перед релизом проверяют, что AI-агенты действительно отказываются нарушать.
6. **Файл CONTEXT-LICENSE** — чётко определяет область действия CC-BY-SA 4.0.

О способах участия см. [руководство по вкладу](../CONTRIBUTING.md), полную техническую спецификацию — [подробности защиты лицензии](../.users/context/contributing.md).

## Быстрый старт

### Предварительные требования

- Linux (Bazzite, Ubuntu, Fedora и др.)
- Node.js 22+, pnpm 9+
- Rust stable (для сборки Tauri)
- Системные пакеты (Fedora): `webkit2gtk4.1-devel libappindicator-gtk3-devel librsvg2-devel`
- cmake (для сборки whisper.cpp)

### Запуск для разработки

```bash
# Install dependencies
cd shell && pnpm install
cd ../agent && pnpm install

# Run Tauri app (Gateway + Agent auto-spawn)
cd ../shell && pnpm run tauri dev
```

При запуске приложение автоматически:
1. Проверяет работоспособность Naia Gateway — если запущен, переиспользует, иначе автоматически spawn
2. Spawn Agent Core (Node.js, соединение stdio)
3. При закрытии приложения завершает только автоматически spawn-нутый Gateway

### Тесты

```bash
cd shell && pnpm test                # Shell unit tests
cd agent && pnpm test                # Agent unit tests
cd agent && pnpm exec tsc --noEmit   # Type check
cargo test --manifest-path shell/src-tauri/Cargo.toml  # Rust tests

# E2E (Gateway + API key required)
cd shell && pnpm run test:e2e:tauri
```

### Сборка Flatpak

```bash
flatpak install --user flathub org.freedesktop.Platform//24.08 org.freedesktop.Sdk//24.08
flatpak-builder --user --install --force-clean build-dir flatpak/io.nextain.naia.yml
flatpak run io.nextain.naia
```

## Модель безопасности

Naia применяет модель безопасности **Defense in Depth (глубокая защита)**:

| Слой | Защита |
|-------|-----------|

## Система памяти

Naia запоминает между сессиями с помощью **Alpha Memory System** — 4-хранилищевая архитектура по образцу человеческой памяти:

| Хранилище | Аналог | Что хранит |
|-------|--------|----------------|
| **Episodes** | Гиппокамп | Реплики диалога с временными метками |
| **Facts** | Неокортекс | Извлечённые факты, предпочтения, сущности |
| **Reflections** | Базальные ганглии | Стратегии, выученные из прошлых неудач |
| **Working Memory** | Префронтальная кора | Активный контекст текущей сессии |

**Как это работает:**
1. Каждое сообщение оценивается по важности (новизна, релевантность, эмоциональный вес)
2. Контент высокой важности сохраняется как **episode**
3. Каждые 30 минут (через 5 минут при бездействии) episode консолидируется в **fact** (ИИ «оглядывается на тот день»)
4. В начале каждой сессии релевантные fact и episode внедряются в контекст

**Ваша память — ваша:**
- Хранится локально в `~/.naia/memory/alpha-memory.json`
- Не передаётся ни на один сервер — даже Nextain
- В **Настройки → Память** можно в любой момент просмотреть и удалить fact
- Достаточно скопировать файл — и резервная копия готова

Полное руководство пользователя см. в [docs/memory.md](../docs/memory.md).

## Обновления ОС

Naia OS построена на [Bazzite](https://github.com/ublue-os/bazzite) (Fedora Atomic). Обновления **атомарны и безопасны**:

- **Автоматически**: еженедельная пересборка вбирает свежие патчи безопасности и обновления Bazzite
- **Атомарно**: новый образ разворачивается рядом с текущим — при сбое старый образ остаётся нетронутым
- **Откат**: выберите предыдущую версию в меню GRUB для мгновенного восстановления
- **Наш оверлей**: добавляются только пакеты (fcitx5, шрифты) + Naia Shell (Flatpak, песочница) + настройки брендинга — ядро, загрузчик и ядро systemd никогда не затрагиваются

```
Bazzite base update → Weekly auto-rebuild → Container smoke test → ISO rebuild → R2 upload
                                                                 ↘ GHCR push → user bootc update
```

Подробности конвейера обновлений: [`.agents/context/update-pipeline.yaml`](../.agents/context/update-pipeline.yaml)

## Процесс разработки

### Разработка функций (базовый) — разработка на основе issue

```
ISSUE → UNDERSTAND → SCOPE → INVESTIGATE → PLAN → BUILD → REVIEW → E2E → SYNC → COMMIT
```

- **3 обязательных гейта** — на UNDERSTAND, SCOPE, PLAN требуется подтверждение пользователя
- **После одобрения плана** — ИИ выполняет от BUILD до COMMIT непрерывно, без остановок
- **Принцип** — сначала читать код апстрима (никаких догадок). Минимальные изменения. Никогда не ломать работающий код.
- **Коммит** — на английском, `<type>(<scope>): <description>`
- **Форматтер** — Biome (табы, двойные кавычки, точки с запятой)

## AI-Native открытый код

В 2025–2026 годах большинство проектов с открытым кодом *защищаются* от вкладов ИИ. **Naia подходит ровно наоборот**: проект спроектирован так, чтобы вклады с помощью ИИ по умолчанию были высокого качества.

> **«Не защищайтесь от ИИ — проектируйте вместе с ИИ.»**

### Как это работает

```
Person (any language) → AI → Git (English) → AI → Person (any language)
```

- **Пишите issue и PR на своём языке** — ИИ всё переведёт
- **И участники, и мейнтейнеры используют ИИ** — кодинг, ревью, триаж
- **Богатый контекст `.agents/`** углубляет понимание проекта ИИ — чем лучше понимание ИИ, тем выше качество вклада
- **10 типов вклада** — перевод, навыки, функции, баги, код, документация, тесты, дизайн, безопасность, контекст
- **Рабочие журналы на родном языке** — ведите приватный репозиторий на своём языке; история Git проверяется через перевод ИИ

Это не просто политика. Это архитектура. Директория `.agents/`, документация triple-mirror и правила защиты лицензии — всё спроектировано так, чтобы сотрудничество с ИИ было структурой, а не случайностью.

Полная модель: [`open-source-operations.yaml`](../.agents/context/open-source-operations.yaml) | [Report (EN)](../docs/reports/20260307-ai-native-opensource-operations.md) | [Report (KO)](../docs/reports/20260307-ai-native-opensource-operations-ko.md)
