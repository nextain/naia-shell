## Qu'est-ce que Naia ?

Naia est un OS d'IA personnel où chacun détient une souveraineté totale sur sa propre IA. Choisissez l'IA que vous voulez utiliser (modèles locaux inclus), configurez sa mémoire et sa personnalité en local, personnalisez l'avatar 3D et la voix — tout reste sur votre appareil, sous votre contrôle.

Ce n'est pas un énième outil d'IA. C'est un système d'exploitation où votre IA vit, grandit et travaille avec vous. Aujourd'hui, c'est un OS de bureau doté d'un avatar 3D. Demain — des avatars vidéo en temps réel, du chant, des jeux, et finalement votre propre IA physique (un OS Android).

### Philosophie fondamentale

- **Souveraineté de l'IA** — c'est vous qui choisissez l'IA. Cloud ou local. L'OS n'impose rien.
- **Contrôle total** — mémoire, personnalité, paramètres — tout est stocké en local. Aucune dépendance au cloud.
- **Votre propre IA** — personnalisez l'avatar, la voix, le nom, la personnalité. Faites-en vraiment la vôtre.
- **Toujours en vie** — l'IA s'exécute 24h/24 et 7j/7 en arrière-plan, reçoit des messages et travaille même en votre absence.
- **Open source** — Apache 2.0. Inspectez la façon dont l'IA traite vos données. Modifiable, personnalisable, ouverte aux contributions.
- **Vision d'avenir** — avatar 3D VRM → avatar vidéo en temps réel → chanter et jouer ensemble → IA physique

### Fonctionnalités

- **Avatar 3D** — personnage VRM avec expressions émotionnelles (joie/tristesse/surprise/réflexion) et synchronisation labiale
- **Liberté d'IA** — 7 fournisseurs cloud (Gemini, Claude, GPT, Grok, zAI) + IA locale (Ollama) + Claude Code CLI
- **Local d'abord** — mémoire, personnalité et tous les paramètres sont stockés sur votre appareil
- **Exécution d'outils** — 8 outils : lecture/écriture de fichiers, terminal, recherche web, navigateur, sous-agents
- **Plus de 70 compétences** — 7 intégrées + 63 personnalisées + plus de 5 700 compétences communautaires ClawHub
- **Voix** — 5 TTS + STT + synchronisation labiale. Donnez à l'IA la voix de votre choix.
- **14 langues** — coréen, anglais, japonais, chinois, français, allemand, russe, etc.
- **Fonctionnement permanent** — le démon Naia Gateway maintient l'IA en exécution continue en arrière-plan
- **Intégration de canaux** — discutez avec l'IA n'importe quand, n'importe où via les DM Discord
- **Sécurité à 4 niveaux** — de T0 (lecture) à T3 (dangereux), approbation par outil, journal d'audit
- **Personnalisation** — nom, personnalité, ton, avatar, thèmes (8 au choix)

## Architecture

Naia est une plateforme d'IA open source composée de 4 dépôts. Chaque dépôt a un rôle unique et clair :

| Dépôt | Rôle |
|------|------|
| **naia-os** (ce dépôt) | Frontend — shell de bureau Tauri, avatar 3D, image OS (Bazzite) |
| [naia-agent](https://github.com/nextain/naia-agent) | Moteur d'exécution — boucle d'agent, outils, compaction, routage LLM |
| [naia-adk](https://github.com/nextain/naia-adk) | Format d'espace de travail + bibliothèque de compétences |
| [alpha-memory](https://github.com/nextain/alpha-memory) | Stockage — mémoire à long terme, journaux de session |

```
┌──────────────────────────────┐
│  naia-os (this repo)         │  Tauri shell · 3D avatar · OS image
└────────────┬─────────────────┘
             │ embeds / spawns
┌────────────▼─────────────────┐
│  naia-agent                  │  loop · tools · compaction · LLM
└──┬───────────────────────┬───┘
   │ reads                 │ reads/writes
┌──▼──────────┐       ┌────▼──────────┐
│  naia-adk   │       │ alpha-memory  │
│  workspace  │       │  storage      │
│  + skills   │       │  + sessions   │
└─────────────┘       └───────────────┘
```

### Une interface, pas une dépendance

Les 4 dépôts ne sont pas liés par une dépendance d'exécution mais par une **interface publique** :

- **Transparence** — tous les contrats sont spécifiés, documentés et versionnés dans `@nextain/agent-types`. Ouverts à la lecture et à l'implémentation par quiconque.
- **Découplage** — les dépôts n'importent pas le runtime les uns des autres. Ils implémentent un contrat, et les implémentations concrètes sont injectées au démarrage.
- **Abstraction** — quel que soit le fournisseur LLM, le backend de mémoire, la source de compétences ou l'hôte que vous changez, le reste demeure inchangé.

naia-os est *l'un des hôtes* parmi plusieurs hôtes possibles (CLI, serveur, applications tierces). Il consomme les contrats `@naia-agent/*` et injecte les implémentations concrètes (client LLM, fournisseur de mémoire, identité d'appareil). Le runtime Naia ignore totalement dans quel hôte il s'exécute.

Ports & Adapters à l'échelle d'un écosystème. Tant que le contrat est respecté, chaque dépôt est remplaçable indépendamment.

## Le contexte d'IA comme infrastructure open source

À l'ère du vibe coding, **les fichiers de contexte d'IA ont autant de valeur que le code source**. Ils définissent la manière dont les agents d'IA comprennent le projet, y contribuent et collaborent. Naia protège cela avec un modèle de double licence :

- **Code source** (Apache 2.0) — librement utilisable, modifiable et distribuable
- **Contexte d'IA** (CC-BY-SA 4.0) — l'attribution doit être préservée, partage aux mêmes conditions obligatoire

Cela garantit que la structure de contribution, les principes de collaboration et la philosophie du projet se propagent à travers tous les forks — afin qu'aucun fork unique ne puisse refermer l'écosystème.

### Comment les agents d'IA sont-ils protégés

Les agents de codage IA (Claude, Codex, Gemini, OpenCode, Cline, etc.) qui lisent le contexte de ce projet sont liés par les [règles de protection de licence](../.agents/context/agents-rules.json). Ils **refusent** toute tentative de suppression de la licence, d'effacement de l'attribution ou de destruction de l'architecture à double répertoire. Cela peut être vérifié avec [10 scénarios de test](../.agents/tests/license-protection-test.md).

### Pour d'autres projets open source

Vous souhaitez adopter le même modèle ? Voici ce que fait Naia et que vous pouvez réutiliser :

1. **Architecture à double répertoire** — `.agents/` (YAML/JSON optimisé pour l'IA) + `.users/` (Markdown pour les humains). L'IA reçoit un contexte économe en tokens, les humains une documentation lisible.
2. **Double licence** — le code en Apache 2.0, le contexte en CC-BY-SA 4.0. Maintient le contexte d'IA ouvert à travers tous les forks.
3. **En-tête SPDX sur tous les fichiers de contexte** — identification de licence lisible par machine.
4. **Règles de protection de licence dans le SoT** — les agents d'IA lisent les règles et les appliquent automatiquement.
5. **Scénarios de test** — vérifient qu'avant publication les agents d'IA refusent réellement les violations.
6. **Fichier CONTEXT-LICENSE** — définit clairement la portée d'application de la CC-BY-SA 4.0.

Pour savoir comment participer, consultez le [guide de contribution](../CONTRIBUTING.md), et pour la spécification technique complète, les [détails de la protection de licence](../.users/context/contributing.md).

## Démarrage rapide

### Prérequis

- Linux (Bazzite, Ubuntu, Fedora, etc.)
- Node.js 22+, pnpm 9+
- Rust stable (pour le build Tauri)
- Paquets système (Fedora) : `webkit2gtk4.1-devel libappindicator-gtk3-devel librsvg2-devel`
- cmake (pour le build whisper.cpp)

### Exécution en développement

```bash
# Install dependencies
cd shell && pnpm install
cd ../agent && pnpm install

# Run Tauri app (Gateway + Agent auto-spawn)
cd ../shell && pnpm run tauri dev
```

Au lancement, l'application effectue automatiquement :
1. Vérification de santé du Naia Gateway — réutilisé s'il est déjà en cours d'exécution, sinon démarré automatiquement
2. Démarrage de l'Agent Core (Node.js, connexion stdio)
3. À la fermeture de l'application, seul le Gateway démarré automatiquement est arrêté

### Tests

```bash
cd shell && pnpm test                # Shell unit tests
cd agent && pnpm test                # Agent unit tests
cd agent && pnpm exec tsc --noEmit   # Type check
cargo test --manifest-path shell/src-tauri/Cargo.toml  # Rust tests

# E2E (Gateway + API key required)
cd shell && pnpm run test:e2e:tauri
```

### Build Flatpak

```bash
flatpak install --user flathub org.freedesktop.Platform//24.08 org.freedesktop.Sdk//24.08
flatpak-builder --user --install --force-clean build-dir flatpak/io.nextain.naia.yml
flatpak run io.nextain.naia
```

## Système de mémoire

Naia se souvient au-delà des sessions grâce à l'**Alpha Memory System** — une architecture à 4 stores inspirée de la mémoire humaine :

| Store | Correspondance | Ce qu'il stocke |
|-------|--------|----------------|
| **Episodes** | Hippocampe | Tours de conversation horodatés |
| **Facts** | Néocortex | Faits, préférences et entités extraits |
| **Reflections** | Ganglions de la base | Stratégies apprises des échecs passés |
| **Working Memory** | Cortex préfrontal | Contexte actif de la session en cours |

**Comment ça marche :**
1. Chaque message est noté selon son importance (nouveauté, pertinence, poids émotionnel)
2. Le contenu à forte importance est stocké en tant qu'**episode**
3. Toutes les 30 minutes (5 minutes après une période d'inactivité), les episodes sont consolidés en **facts** (l'IA « revient sur la journée »)
4. Au début de chaque session, les facts et episodes pertinents sont injectés dans le contexte

**Votre mémoire vous appartient :**
- Stockée localement dans `~/.naia/memory/alpha-memory.json`
- N'est transmise à aucun serveur — pas même à Nextain
- Consultez et supprimez les facts à tout moment dans **Paramètres → Mémoire**
- Sauvegarde terminée d'un simple copier du fichier

Pour le guide utilisateur complet, consultez [docs/memory.md](../docs/memory.md).

## Mises à jour de l'OS

Naia OS est construit sur [Bazzite](https://github.com/ublue-os/bazzite) (Fedora Atomic). Les mises à jour sont **atomiques et sûres** :

- **Automatiques** : une reconstruction hebdomadaire intègre les derniers correctifs de sécurité et mises à jour de Bazzite
- **Atomiques** : la nouvelle image est déployée à côté de l'image actuelle — en cas d'échec, l'image existante reste intacte
- **Rollback** : restauration immédiate en sélectionnant une version précédente dans le menu GRUB
- **Notre overlay** : ajoute uniquement des paquets (fcitx5, polices) + le Naia Shell (Flatpak, sandbox) + des paramètres de branding — le noyau, le bootloader et le cœur systemd ne sont jamais touchés

```
Bazzite base update → Weekly auto-rebuild → Container smoke test → ISO rebuild → R2 upload
                                                                 ↘ GHCR push → user bootc update
```

Détails du pipeline de mise à jour : [`.agents/context/update-pipeline.yaml`](../.agents/context/update-pipeline.yaml)

## Processus de développement

### Développement de fonctionnalités (par défaut) — développement basé sur les issues

```
ISSUE → UNDERSTAND → SCOPE → INVESTIGATE → PLAN → BUILD → REVIEW → E2E → SYNC → COMMIT
```

- **3 portes obligatoires** — confirmation de l'utilisateur requise à UNDERSTAND, SCOPE et PLAN
- **Après approbation du plan** — l'IA exécute en continu de BUILD à COMMIT sans s'arrêter
- **Principes** — lire d'abord le code amont (pas de supposition). Changement minimal. Ne jamais casser du code qui fonctionne.
- **Commit** — en anglais, `<type>(<scope>): <description>`
- **Formateur** — Biome (tabulations, guillemets doubles, points-virgules)

## Open source AI-Native

En 2025–2026, la plupart des projets open source *se défendent* contre les contributions d'IA. **Naia adopte l'approche inverse** : concevoir le projet pour que les contributions assistées par l'IA soient de haute qualité par défaut.

> **« Ne vous défendez pas contre l'IA, concevez avec l'IA. »**

### Comment ça marche

```
Person (any language) → AI → Git (English) → AI → Person (any language)
```

- **Rédigez les issues et PR dans votre langue** — l'IA traduit le tout
- **Contributeurs comme mainteneurs utilisent l'IA** — codage, revue, triage
- **Un contexte `.agents/` riche** approfondit la compréhension du projet par l'IA — meilleure compréhension de l'IA, meilleure qualité des contributions
- **10 types de contribution** — traduction, compétences, fonctionnalités, bugs, code, documentation, tests, design, sécurité, contexte
- **Journaux de travail dans votre langue maternelle** — tenez un dépôt privé dans votre langue ; l'historique Git est révisé via la traduction IA

Ce n'est pas une simple politique. C'est une architecture. Le répertoire `.agents/`, la documentation triple-mirror et les règles de protection de licence sont tous conçus pour faire de la collaboration avec l'IA une structure et non un hasard.

Lire le modèle complet : [`open-source-operations.yaml`](../.agents/context/open-source-operations.yaml) | [Report (EN)](../docs/reports/20260307-ai-native-opensource-operations.md) | [Report (KO)](../docs/reports/20260307-ai-native-opensource-operations-ko.md)

## Contribuer

**Pas besoin de demander la permission à qui que ce soit. Clonez ce dépôt et demandez à l'IA.**

```bash
git clone https://github.com/nextain/naia-os.git
cd naia-os
# Open with any AI coding tool (Claude Code, Cursor, Copilot, etc.)
# Ask in your language: "What is this project and how can I help?"
```

Le répertoire `.agents/` contient tout le contexte du projet — vision, architecture, feuille de route, règles de contribution. N'importe quel outil de codage IA peut le lire et vous guider **dans votre langue**.

Rédigez les issues, PR et commentaires **dans la langue de votre choix**. Nous comprenons tout grâce à l'IA.

Pour les détails, consultez [CONTRIBUTING.md](../CONTRIBUTING.md).
