## Naia কী?

Naia একটি ব্যক্তিগত AI OS, যেখানে প্রতিটি ব্যক্তি তার নিজের AI-এর উপর সম্পূর্ণ সার্বভৌমত্ব রাখে। কোন AI ব্যবহার করবেন তা বেছে নিন (লোকাল মডেল সহ), তার স্মৃতি ও ব্যক্তিত্ব লোকালভাবে সেট করুন, এবং 3D অবতার ও কণ্ঠ কাস্টমাইজ করুন — সবকিছু আপনার ডিভাইসে, আপনার নিয়ন্ত্রণে থাকে।

এটি আরেকটি AI টুল নয়। এটি একটি অপারেটিং সিস্টেম, যেখানে আপনার AI বাস করে, বেড়ে ওঠে এবং আপনার সাথে কাজ করে। আজ এটি একটি 3D অবতারসহ ডেস্কটপ OS। আগামীকাল — রিয়েল-টাইম ভিডিও অবতার, গান, গেম, এবং অবশেষে আপনার নিজস্ব Physical AI (অ্যান্ড্রয়েড OS) হয়ে।

### মূল দর্শন

- **AI সার্বভৌমত্ব** — AI আপনি বেছে নেন। ক্লাউড হোক বা লোকাল। OS তা চাপিয়ে দেয় না।
- **সম্পূর্ণ নিয়ন্ত্রণ** — স্মৃতি, ব্যক্তিত্ব, সেটিংস — সবই লোকালভাবে সংরক্ষিত। কোনো ক্লাউড নির্ভরতা নেই।
- **আপনার নিজস্ব AI** — অবতার, কণ্ঠ, নাম, ব্যক্তিত্ব কাস্টমাইজ করুন। সত্যিকারভাবে আপনার করে নিন।
- **সর্বদা সচল** — AI ব্যাকগ্রাউন্ডে ২৪/৭ চলে, আপনি দূরে থাকলেও বার্তা গ্রহণ করে এবং কাজ করে।
- **ওপেন সোর্স** — Apache 2.0। AI আপনার ডেটা কীভাবে পরিচালনা করে তা পরিদর্শন করুন। সংশোধন·কাস্টমাইজ·অবদান সম্ভব।
- **ভবিষ্যৎ দৃষ্টিভঙ্গি** — VRM 3D অবতার → রিয়েল-টাইম ভিডিও অবতার → একসাথে গান·গেম → Physical AI

### বৈশিষ্ট্য

- **3D অবতার** — আবেগ প্রকাশ (আনন্দ/দুঃখ/বিস্ময়/চিন্তা) এবং লিপ-সিঙ্কসহ VRM চরিত্র
- **AI স্বাধীনতা** — ৭টি ক্লাউড প্রোভাইডার (Gemini, Claude, GPT, Grok, zAI) + লোকাল AI (Ollama) + Claude Code CLI
- **লোকাল-প্রথম** — স্মৃতি, ব্যক্তিত্ব, সমস্ত সেটিংস আপনার ডিভাইসে সংরক্ষিত
- **টুল এক্সিকিউশন** — ৮টি টুল: ফাইল পড়া/লেখা, টার্মিনাল, ওয়েব সার্চ, ব্রাউজার, সাব-এজেন্ট
- **৭০+ স্কিল** — অন্তর্নির্মিত ৭ + কাস্টম ৬৩ + ClawHub কমিউনিটি স্কিল ৫,৭০০+
- **কণ্ঠ** — ৫ ধরনের TTS + STT + লিপ-সিঙ্ক। আপনার পছন্দের কণ্ঠ AI-কে দিন।
- **১৪টি ভাষা** — কোরিয়ান, ইংরেজি, জাপানি, চীনা, ফরাসি, জার্মান, রুশ ইত্যাদি
- **সর্বদা চালু** — Naia Gateway ডেমন AI-কে ব্যাকগ্রাউন্ডে অবিরত চালিয়ে রাখে
- **চ্যানেল সংযোগ** — Discord DM-এর মাধ্যমে যেকোনো সময় যেকোনো জায়গায় AI-এর সাথে কথা বলুন
- **৪-স্তর নিরাপত্তা** — T0 (পড়া) থেকে T3 (ঝুঁকিপূর্ণ) পর্যন্ত, টুলভিত্তিক অনুমোদন, অডিট লগ
- **ব্যক্তিগতকরণ** — নাম, ব্যক্তিত্ব, কথার ধরন, অবতার, থিম (৮ ধরনের)

## কেন Naia?

অন্যান্য AI টুল শুধুই "টুল"। Naia হলো **"আপনার নিজস্ব AI"**।

| | অন্যান্য AI টুল | Naia |
|---|----------------|------|
| **দর্শন** | AI-কে টুল হিসেবে ব্যবহার | AI-কে একটি OS দেওয়া হয়। একসাথে বাস করে। |
| **লক্ষ্য** | শুধুমাত্র ডেভেলপার | যারা নিজের AI চান, তাদের সবাই |
| **AI পছন্দ** | প্ল্যাটফর্ম ঠিক করে | ক্লাউড ৭ + লোকাল AI — আপনি ঠিক করেন |
| **ডেটা** | ক্লাউড-নির্ভর | স্মৃতি·ব্যক্তিত্ব·সেটিংস সবই লোকাল |
| **অবতার** | নেই | VRM 3D চরিত্র + আবেগ + লিপ-সিঙ্ক |
| **কণ্ঠ** | শুধু টেক্সট অথবা মৌলিক TTS | ৫ TTS + STT + AI-এর নিজস্ব কণ্ঠ |
| **বিতরণ** | npm / brew / pip | ডেস্কটপ অ্যাপ অথবা বুটযোগ্য USB OS |
| **প্ল্যাটফর্ম** | macOS / CLI / Web | Linux নেটিভ ডেস্কটপ → ভবিষ্যৎ: Physical AI |
| **খরচ** | আলাদা API কী প্রয়োজন | শুরুতে ফ্রি ক্রেডিট, লোকাল AI সম্পূর্ণ বিনামূল্যে |

## OpenClaw-এর সাথে সম্পর্ক

Naia [OpenClaw](https://github.com/openclaw-ai/openclaw) ইকোসিস্টেমের উপর নির্মিত, তবে মৌলিকভাবে একটি ভিন্ন পণ্য।

| | OpenClaw | Naia |
|---|---------|---------|
| **রূপ** | CLI ডেমন + টার্মিনাল | ডেস্কটপ অ্যাপ + 3D অবতার |
| **লক্ষ্য** | ডেভেলপার | সবাই |
| **UI** | নেই (টার্মিনাল) | Tauri 2 নেটিভ অ্যাপ (React + Three.js) |
| **অবতার** | নেই | VRM 3D চরিত্র (আবেগ, লিপ-সিঙ্ক, দৃষ্টি) |
| **LLM** | একক প্রোভাইডার | মাল্টি প্রোভাইডার ৭ + রিয়েল-টাইম স্যুইচিং |
| **কণ্ঠ** | TTS 3 (Edge, OpenAI, ElevenLabs) | TTS 5 (+Google, Nextain) + STT + অবতার লিপ-সিঙ্ক |
| **আবেগ** | নেই | অভিব্যক্তিতে ম্যাপ করা ৬ ধরনের আবেগ |
| **অনবোর্ডিং** | CUI | GUI + VRM অবতার নির্বাচন |
| **খরচ ট্র্যাকিং** | নেই | রিয়েল-টাইম ক্রেডিট ড্যাশবোর্ড |
| **বিতরণ** | npm install | Flatpak / AppImage / DEB / RPM + OS ইমেজ |
| **বহুভাষিক** | ইংরেজি CLI | ১৪টি ভাষার GUI |
| **চ্যানেল** | সার্ভার বট (মাল্টিচ্যানেল) | Naia-ডেডিকেটেড Discord DM বট |

**OpenClaw থেকে নেওয়া:** ডেমন আর্কিটেকচার, টুল এক্সিকিউশন ইঞ্জিন, চ্যানেল সিস্টেম, স্কিল ইকোসিস্টেম (৫,৭০০+ Clawhub স্কিল সামঞ্জস্যপূর্ণ)

**Naia যা নতুন করে তৈরি করেছে:** Tauri Shell, VRM অবতার সিস্টেম, মাল্টি LLM এজেন্ট, আবেগ ইঞ্জিন, TTS/STT ইন্টিগ্রেশন, অনবোর্ডিং উইজার্ড, খরচ ট্র্যাকিং, Nextain অ্যাকাউন্ট সংযোগ, Alpha Memory System, নিরাপত্তা স্তর

### নির্ভরতা নয়, ইন্টারফেস

৪টি রেপো রানটাইম নির্ভরতা নয়, বরং **প্রকাশিত ইন্টারফেস** দিয়ে যুক্ত:

- **স্বচ্ছতা** — সমস্ত চুক্তি `@nextain/agent-types`-এ নির্দিষ্ট·নথিভুক্ত·সংস্করণ-পরিচালিত। যে কেউ পড়তে বা বাস্তবায়ন করতে উন্মুক্ত।
- **অসংযুক্তি** — রেপোগুলো একে অপরের রানটাইম import করে না। চুক্তি বাস্তবায়ন করে, এবং নির্দিষ্ট বাস্তবায়ন শুরুতে ইনজেক্ট করা হয়।
- **বিমূর্ততা** — যেকোনো LLM প্রোভাইডার, মেমরি ব্যাকএন্ড, স্কিল সোর্স বা হোস্ট বদলালেও বাকিগুলো অপরিবর্তিত থাকে।

naia-os হলো সম্ভাব্য একাধিক হোস্টের (CLI, সার্ভার, থার্ড-পার্টি অ্যাপ) মধ্যে *একটি হোস্ট*। এটি `@naia-agent/*` চুক্তি গ্রহণ করে এবং নির্দিষ্ট বাস্তবায়ন (LLM ক্লায়েন্ট, মেমরি প্রোভাইডার, ডিভাইস পরিচয়) ইনজেক্ট করে। Naia রানটাইম জানে না সে কোন হোস্টের ভিতরে চলছে।

ইকোসিস্টেম-স্কেলের Ports & Adapters। চুক্তি মেনে চললে প্রতিটি রেপো স্বাধীনভাবে প্রতিস্থাপনযোগ্য।

## ওপেন সোর্স অবকাঠামো হিসেবে AI কনটেক্সট

ভাইব কোডিং যুগে **AI কনটেক্সট ফাইল সোর্স কোডের মতোই মূল্যবান**। এটি নির্ধারণ করে কীভাবে AI এজেন্ট প্রকল্পটি বোঝে, অবদান রাখে এবং সহযোগিতা করে। Naia এটিকে একটি ডুয়াল-লাইসেন্স মডেল দিয়ে রক্ষা করে:

- **সোর্স কোড** (Apache 2.0) — স্বাধীনভাবে ব্যবহার·সংশোধন·বিতরণ
- **AI কনটেক্সট** (CC-BY-SA 4.0) — সূত্র উল্লেখ সংরক্ষণ, একই শর্তে শেয়ার করা আবশ্যক

এটি নিশ্চিত করে যে অবদান কাঠামো, সহযোগিতার নীতি এবং প্রকল্পের দর্শন প্রতিটি ফর্কের মাধ্যমে ছড়িয়ে পড়ে — যাতে কোনো একক ফর্ক ইকোসিস্টেমকে বন্ধ করতে না পারে।

### AI এজেন্ট কীভাবে রক্ষিত হয়

এই প্রকল্পের কনটেক্সট পড়া AI কোডিং এজেন্ট (Claude, Codex, Gemini, OpenCode, Cline ইত্যাদি) [লাইসেন্স সুরক্ষা নিয়ম](../.agents/context/agents-rules.json) দ্বারা আবদ্ধ। লাইসেন্স অপসারণ, সূত্র মুছে ফেলা, ডুয়াল-ডিরেক্টরি আর্কিটেকচার ধ্বংসের চেষ্টা **প্রত্যাখ্যান** করে। [১০টি টেস্ট সিনারিও](../.agents/tests/license-protection-test.md) দিয়ে এটি যাচাই করা যায়।

### অন্যান্য ওপেন সোর্স প্রকল্পের জন্য

একই প্যাটার্ন গ্রহণ করতে চান? Naia যা করে তার মধ্যে যা পুনর্ব্যবহারযোগ্য:

1. **ডুয়াল-ডিরেক্টরি আর্কিটেকচার** — `.agents/` (AI-অপ্টিমাইজড YAML/JSON) + `.users/` (মানুষের জন্য Markdown)। AI পায় টোকেন-দক্ষ কনটেক্সট, মানুষ পায় সহজপাঠ্য নথি।
2. **ডুয়াল লাইসেন্স** — কোড Apache 2.0, কনটেক্সট CC-BY-SA 4.0। ফর্ক জুড়ে AI কনটেক্সট উন্মুক্ত রাখে।
3. **প্রতিটি কনটেক্সট ফাইলে SPDX হেডার** — মেশিন-পাঠযোগ্য লাইসেন্স শনাক্তকরণ।
4. **SoT-এর লাইসেন্স সুরক্ষা নিয়ম** — AI এজেন্ট নিয়ম পড়ে এবং স্বয়ংক্রিয়ভাবে প্রয়োগ করে।
5. **টেস্ট সিনারিও** — রিলিজের আগে যাচাই করুন AI এজেন্ট আসলেই লঙ্ঘন প্রত্যাখ্যান করে কিনা।
6. **CONTEXT-LICENSE ফাইল** — CC-BY-SA 4.0 কোথায় প্রযোজ্য তা স্পষ্টভাবে সংজ্ঞায়িত।

### ডেভেলপমেন্ট রান

```bash
# Install dependencies
cd shell && pnpm install
cd ../agent && pnpm install

```

অ্যাপ চালু হলে স্বয়ংক্রিয়ভাবে:

3. অ্যাপ বন্ধ হলে শুধু স্বয়ংক্রিয়ভাবে spawn করা Gateway বন্ধ হয়

## মেমরি সিস্টেম

Naia **Alpha Memory System** দিয়ে সেশন পেরিয়েও মনে রাখে — মানব স্মৃতির আদলে তৈরি একটি 4-store আর্কিটেকচার:

| স্টোর | সমতুল্য | যা সংরক্ষণ করে |
|-------|--------|----------------|
| **Episodes** | হিপোক্যাম্পাস | টাইমস্ট্যাম্প করা কথোপকথনের টার্ন |
| **Facts** | নিওকর্টেক্স | নিষ্কাশিত তথ্য, পছন্দ, এনটিটি |
| **Reflections** | বেসাল গ্যাংলিয়া | অতীত ব্যর্থতা থেকে শেখা কৌশল |
| **Working Memory** | প্রিফ্রন্টাল কর্টেক্স | বর্তমান সেশনের সক্রিয় কনটেক্সট |

**কীভাবে কাজ করে:**
1. প্রতিটি বার্তা গুরুত্ব (নতুনত্ব, প্রাসঙ্গিকতা, আবেগের ওজন) অনুসারে স্কোর করা হয়
2. উচ্চ গুরুত্বের কনটেন্ট **episode** হিসেবে সংরক্ষিত হয়
3. প্রতি ৩০ মিনিটে (নিষ্ক্রিয় থাকলে ৫ মিনিট পর) episode **fact**-এ একীভূত হয় (AI "সেদিনের দিকে ফিরে তাকায়")
4. প্রতিটি সেশনের শুরুতে প্রাসঙ্গিক fact ও episode কনটেক্সট হিসেবে ইনজেক্ট করা হয়

**আপনার স্মৃতি আপনার নিজের:**
- `~/.naia/memory/alpha-memory.json`-এ লোকালভাবে সংরক্ষিত
- কোনো সার্ভারে পাঠানো হয় না — Nextain-এও নয়
- **সেটিংস → মেমরি**-তে যেকোনো সময় fact দেখুন·মুছুন
- শুধু ফাইলটি কপি করলেই ব্যাকআপ সম্পূর্ণ

সম্পূর্ণ ব্যবহারকারী নির্দেশিকার জন্য [docs/memory.md](../docs/memory.md) দেখুন।

## OS আপডেট

Naia OS [Bazzite](https://github.com/ublue-os/bazzite) (Fedora Atomic) এর উপর নির্মিত। আপডেট **পারমাণবিক ও নিরাপদ**:

- **স্বয়ংক্রিয়**: সাপ্তাহিক রিবিল্ড সর্বশেষ Bazzite নিরাপত্তা প্যাচ ও আপডেট প্রতিফলিত করে
- **পারমাণবিক**: নতুন ইমেজ বর্তমান ইমেজের পাশাপাশি স্থাপন করা হয় — ব্যর্থ হলে বিদ্যমান ইমেজ অপরিবর্তিত থাকে
- **রোলব্যাক**: GRUB মেনু থেকে পূর্বের সংস্করণ বেছে নিয়ে তাৎক্ষণিক পুনরুদ্ধার
- **আমাদের ওভারলে**: শুধু প্যাকেজ (fcitx5, ফন্ট) + Naia Shell (Flatpak, স্যান্ডবক্স) + ব্র্যান্ডিং সেটিংস যোগ করে — কার্নেল·বুটলোডার·systemd কোর কখনো স্পর্শ করে না

```
Bazzite base update → Weekly auto-rebuild → Container smoke test → ISO rebuild → R2 upload
                                                                 ↘ GHCR push → user bootc update
```

আপডেট পাইপলাইন বিস্তারিত: [`.agents/context/update-pipeline.yaml`](../.agents/context/update-pipeline.yaml)

## ডেভেলপমেন্ট প্রক্রিয়া

### ফিচার ডেভেলপমেন্ট (মৌলিক) — ইস্যু-ভিত্তিক ডেভেলপমেন্ট

```
ISSUE → UNDERSTAND → SCOPE → INVESTIGATE → PLAN → BUILD → REVIEW → E2E → SYNC → COMMIT
```

- **৩টি বাধ্যতামূলক গেট** — UNDERSTAND, SCOPE, PLAN-এ ব্যবহারকারীর নিশ্চিতকরণ প্রয়োজন
- **পরিকল্পনা অনুমোদনের পর** — AI BUILD থেকে COMMIT পর্যন্ত না থেমে অবিরত চালায়
- **নীতি** — আগে আপস্ট্রিম কোড পড়ুন (অনুমান নিষেধ)। ন্যূনতম পরিবর্তন। কাজ করা কোড কখনো ভাঙবেন না।
- **কমিট** — ইংরেজি, `<type>(<scope>): <description>`
- **ফরম্যাটার** — Biome (ট্যাব, ডাবল কোট, সেমিকোলন)

## AI-Native ওপেন সোর্স

২০২৫–২০২৬ সালে বেশিরভাগ ওপেন সোর্স প্রকল্প AI অবদানের বিরুদ্ধে *প্রতিরক্ষা* করছে। **Naia ঠিক বিপরীত পথে চলে**: প্রকল্পটি এমনভাবে ডিজাইন করা হয়েছে যাতে AI-সহায়তায় করা অবদান ডিফল্টভাবেই উচ্চমানের হয়।

> **"AI-এর বিরুদ্ধে প্রতিরক্ষা নয়, AI-এর সাথে ডিজাইন করুন।"**

### কীভাবে কাজ করে

```
Person (any language) → AI → Git (English) → AI → Person (any language)
```

- **আপনার ভাষায় ইস্যু ও PR লিখুন** — AI সব অনুবাদ করে
- **অবদানকারী ও মেইনটেইনার উভয়েই AI ব্যবহার করে** — কোডিং, রিভিউ, ট্রায়াজ
- **সমৃদ্ধ `.agents/` কনটেক্সট** AI-এর প্রকল্প-বোঝাপড়াকে গভীর করে — AI যত ভালো বোঝে, অবদানের মান তত বাড়ে↑
- **১০ ধরনের অবদান** — অনুবাদ, স্কিল, ফিচার, বাগ, কোড, নথি, টেস্ট, ডিজাইন, নিরাপত্তা, কনটেক্সট
- **কাজের লগ মাতৃভাষায়** — নিজের ভাষায় প্রাইভেট রেপো রাখুন; Git ইতিহাস AI অনুবাদে পর্যালোচনা করা হয়

এটি কেবল একটি নীতি নয়। এটি একটি আর্কিটেকচার। `.agents/` ডিরেক্টরি, triple-mirror নথি, লাইসেন্স সুরক্ষা নিয়ম — সবই AI সহযোগিতাকে দৈবক্রমে নয়, কাঠামোগতভাবে গড়ে তুলতে ডিজাইন করা হয়েছে।

## অবদান

**কারও অনুমতির প্রয়োজন নেই। এই রেপো ক্লোন করুন এবং AI-কে জিজ্ঞাসা করুন।**

```bash
git clone https://github.com/nextain/naia-os.git
cd naia-os
# Open with any AI coding tool (Claude Code, Cursor, Copilot, etc.)
# Ask in your language: "What is this project and how can I help?"
```

`.agents/` ডিরেক্টরিতে সম্পূর্ণ প্রকল্প কনটেক্সট — দৃষ্টিভঙ্গি, আর্কিটেকচার, রোডম্যাপ, অবদান নিয়ম — রয়েছে। যেকোনো AI কোডিং টুল এটি পড়ে **আপনার ভাষায়** পথ দেখাতে পারে।

ইস্যু, PR, কমেন্ট **যেকোনো ভাষায়** লিখুন। আমরা AI দিয়ে সবকিছু বুঝি।
