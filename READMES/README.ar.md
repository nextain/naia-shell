## ما هي Naia؟

Naia هي نظام تشغيل ذكاء اصطناعي شخصي يمتلك فيه الفرد سيادة كاملة على ذكائه الاصطناعي. اختر أي ذكاء اصطناعي تستخدمه (بما في ذلك النماذج المحلية)، واضبط ذاكرته وشخصيته محليًا، وخصّص الأفاتار ثلاثي الأبعاد والصوت — كل شيء يبقى على جهازك، تحت سيطرتك.

هذه ليست مجرد أداة ذكاء اصطناعي أخرى. إنها نظام تشغيل يعيش فيه ذكاؤك الاصطناعي وينمو ويعمل معك. اليوم هي نظام تشغيل سطح مكتب بأفاتار ثلاثي الأبعاد. وغدًا — أفاتار فيديو في الزمن الحقيقي، وغناء، وألعاب، وفي النهاية ذكاء اصطناعي فيزيائي خاص بك (نظام تشغيل أندرويد).

### الفلسفة الأساسية

- **سيادة الذكاء الاصطناعي** — أنت من يختار الذكاء الاصطناعي. سحابيًا كان أم محليًا. لا يفرضه نظام التشغيل.
- **سيطرة كاملة** — الذاكرة، الشخصية، الإعدادات — كلها مخزّنة محليًا. لا اعتماد على السحابة.
- **ذكاؤك الاصطناعي الخاص** — خصّص الأفاتار والصوت والاسم والشخصية. اجعله ملكك حقًا.
- **حيٌّ دائمًا** — يعمل الذكاء الاصطناعي في الخلفية على مدار الساعة طوال الأسبوع، ويستقبل الرسائل ويعمل حتى حين تكون غائبًا.
- **مفتوح المصدر** — Apache 2.0. افحص كيف يتعامل الذكاء الاصطناعي مع بياناتك. قابل للتعديل والتخصيص والمساهمة.
- **رؤية المستقبل** — أفاتار VRM ثلاثي الأبعاد ← أفاتار فيديو في الزمن الحقيقي ← الغناء واللعب معًا ← ذكاء اصطناعي فيزيائي

### المزايا

- **أفاتار ثلاثي الأبعاد** — شخصية VRM بتعبير عن المشاعر (فرح/حزن/دهشة/تفكير) ومزامنة شفاه
- **حرية الذكاء الاصطناعي** — 7 مزوّدين سحابيين (Gemini, Claude, GPT, Grok, zAI) + ذكاء اصطناعي محلي (Ollama) + Claude Code CLI
- **المحلي أولًا** — الذاكرة والشخصية وكل الإعدادات مخزّنة على جهازك
- **تنفيذ الأدوات** — 8 أدوات: قراءة/كتابة الملفات، الطرفية، البحث على الويب، المتصفح، الوكلاء الفرعيون
- **70+ مهارة** — 7 مدمجة + 63 مخصّصة + 5,700+ مهارة مجتمعية من ClawHub
- **الصوت** — 5 أنواع TTS + STT + مزامنة شفاه. ألبس ذكاءك الاصطناعي الصوت الذي تريد.
- **14 لغة** — الكورية، الإنجليزية، اليابانية، الصينية، الفرنسية، الألمانية، الروسية وغيرها
- **تشغيل دائم** — يُبقي خادم Naia Gateway الذكاءَ الاصطناعي يعمل في الخلفية باستمرار
- **ربط القنوات** — تحدّث مع ذكائك الاصطناعي في أي وقت ومكان عبر رسائل Discord المباشرة
- **أمان من 4 مستويات** — من T0 (قراءة) إلى T3 (خطر)، موافقة لكل أداة، سجل تدقيق
- **التخصيص** — الاسم، الشخصية، أسلوب الكلام، الأفاتار، السمات (8 أنواع)

## لماذا Naia؟

أدوات الذكاء الاصطناعي الأخرى مجرد "أدوات". أما Naia فهي **"ذكاؤك الاصطناعي الخاص"**.

| | أدوات ذكاء اصطناعي أخرى | Naia |
|---|----------------|------|
| **الفلسفة** | استخدام الذكاء الاصطناعي كأداة | منح الذكاء الاصطناعي نظام تشغيل. العيش معًا. |
| **الجمهور** | للمطوّرين فقط | كل من يريد ذكاءً اصطناعيًا خاصًا به |
| **اختيار الذكاء الاصطناعي** | تقرّره المنصّة | 7 سحابي + ذكاء اصطناعي محلي — أنت من يقرّر |
| **البيانات** | تابعة للسحابة | الذاكرة والشخصية والإعدادات كلها محلية |
| **الأفاتار** | لا يوجد | شخصية VRM ثلاثية الأبعاد + مشاعر + مزامنة شفاه |
| **الصوت** | نص فقط أو TTS أساسي | 5 TTS + STT + صوت خاص بالذكاء الاصطناعي |
| **التوزيع** | npm / brew / pip | تطبيق سطح مكتب أو نظام تشغيل USB قابل للإقلاع |
| **المنصّة** | macOS / CLI / Web | سطح مكتب Linux أصيل ← المستقبل: ذكاء اصطناعي فيزيائي |
| **التكلفة** | يتطلب مفتاح API منفصل | رصيد مجاني للبدء، ذكاء اصطناعي محلي مجاني تمامًا |

| | OpenClaw | Naia |
|---|---------|---------|
| **الشكل** | خادم CLI + طرفية | تطبيق سطح مكتب + أفاتار ثلاثي الأبعاد |
| **الجمهور** | المطوّرون | الجميع |
| **واجهة المستخدم** | لا يوجد (طرفية) | تطبيق Tauri 2 أصيل (React + Three.js) |
| **الأفاتار** | لا يوجد | شخصية VRM ثلاثية الأبعاد (مشاعر، مزامنة شفاه، اتجاه نظر) |
| **LLM** | مزوّد واحد | متعدّد المزوّدين 7 + تبديل في الزمن الحقيقي |
| **الصوت** | TTS 3 (Edge, OpenAI, ElevenLabs) | TTS 5 (+Google, Nextain) + STT + مزامنة شفاه الأفاتار |
| **المشاعر** | لا يوجد | 6 مشاعر مرتبطة بتعابير الوجه |
| **التأهيل (Onboarding)** | CUI | GUI + اختيار أفاتار VRM |
| **تتبّع التكلفة** | لا يوجد | لوحة رصيد في الزمن الحقيقي |
| **التوزيع** | npm install | Flatpak / AppImage / DEB / RPM + صورة نظام تشغيل |
| **تعدّد اللغات** | CLI إنجليزي | واجهة GUI بـ 14 لغة |
| **القنوات** | بوت خادم (متعدّد القنوات) | بوت رسائل Discord المباشرة خاص بـ Naia |

**ما أُخذ من OpenClaw:** بنية الخادم، محرّك تنفيذ الأدوات، نظام القنوات، منظومة المهارات (متوافقة مع 5,700+ مهارة Clawhub)

**ما صنعته Naia جديدًا:** Tauri Shell، نظام أفاتار VRM، وكيل LLM متعدّد، محرّك المشاعر، تكامل TTS/STT، معالج التأهيل، تتبّع التكلفة، ربط حساب Nextain، Alpha Memory System، طبقة الأمان

## البنية المعمارية

Naia منصّة ذكاء اصطناعي مفتوحة المصدر مكوّنة من 4 مستودعات. لكل مستودع دور واحد واضح:

| المستودع | الدور |
|------|------|
| **naia-os** (هذا المستودع) | الواجهة الأمامية — قشرة Tauri لسطح المكتب، أفاتار ثلاثي الأبعاد، صورة نظام التشغيل (Bazzite) |
| [naia-agent](https://github.com/nextain/naia-agent) | محرّك التشغيل — حلقة الوكيل، الأدوات، الضغط (compaction)، توجيه LLM |
| [naia-adk](https://github.com/nextain/naia-adk) | صيغة مساحة العمل + مكتبة المهارات |
| [alpha-memory](https://github.com/nextain/alpha-memory) | التخزين — الذاكرة طويلة الأمد، سجلات الجلسات |

```
┌──────────────────────────────┐
│  naia-os (this repo)         │  Tauri shell · 3D avatar · OS image
└────────────┬─────────────────┘
             │ embeds / spawns
┌────────────▼─────────────────┐
│  naia-agent                  │  loop · tools · compaction · LLM
└──┬───────────────────────┬───┘
   │ reads                 │ reads/writes
┌──▼──────────┐       ┌────▼──────────┐
│  naia-adk   │       │ alpha-memory  │
│  workspace  │       │  storage      │
│  + skills   │       │  + sessions   │
└─────────────┘       └───────────────┘
```

### واجهات لا تبعيّات

المستودعات الأربعة متّصلة عبر **واجهات مُعلَنة** لا عبر تبعيّات وقت التشغيل:

- **الشفافية** — كل العقود مواصَفة وموثّقة ومُدارة الإصدارات في `@nextain/agent-types`. مفتوحة لأي أحد ليقرأها أو ينفّذها.
- **عدم الارتباط** — لا تستورد المستودعات وقت تشغيل بعضها. تنفّذ العقود، وتُحقَن التطبيقات المحدّدة عند البدء.
- **التجريد** — يمكنك تبديل أي مزوّد LLM أو خلفية ذاكرة أو مصدر مهارات أو مضيف، ويبقى الباقي كما هو.

naia-os هي *مضيف واحد* من بين عدة مضيفين ممكنين (CLI، خادم، تطبيق طرف ثالث). تستهلك عقود `@naia-agent/*` وتحقن التطبيقات المحدّدة (عميل LLM، مزوّد ذاكرة، هوية الجهاز). لا يعرف وقت تشغيل Naia إطلاقًا داخل أي مضيف يعمل.

أنماط Ports & Adapters على مستوى المنظومة. ما دامت العقود محفوظة، فكل مستودع قابل للاستبدال باستقلال.

## سياق الذكاء الاصطناعي كبنية تحتية مفتوحة المصدر

في عصر برمجة الـ vibe، **ملفات سياق الذكاء الاصطناعي ذات قيمة بقدر شيفرة المصدر**. فهي تحدّد كيف يفهم وكلاء الذكاء الاصطناعي المشروعَ ويساهمون ويتعاونون. وتحمي Naia ذلك بنموذج ترخيص مزدوج:

- **شيفرة المصدر** (Apache 2.0) — استخدام وتعديل وتوزيع بحرية
- **سياق الذكاء الاصطناعي** (CC-BY-SA 4.0) — الحفاظ على نسبة المصدر، والمشاركة بالشروط نفسها إلزامية

يضمن هذا أن تنتشر بنية المساهمة ومبادئ التعاون وفلسفة المشروع عبر كل تفرّع — وألّا يستطيع أي تفرّع منفرد إغلاق المنظومة.

### كيف يُحمى وكلاء الذكاء الاصطناعي

وكلاء برمجة الذكاء الاصطناعي (Claude, Codex, Gemini, OpenCode, Cline وغيرهم) الذين يقرؤون سياق هذا المشروع مقيَّدون بـ[قواعد حماية الترخيص](../.agents/context/agents-rules.json). فهم **يرفضون** محاولات إزالة الترخيص أو حذف نسبة المصدر أو تخريب بنية الدليل المزدوج. ويمكن التحقق من ذلك عبر [10 سيناريوهات اختبار](../.agents/tests/license-protection-test.md).

### لمشاريع مفتوحة المصدر أخرى

أترغب في تبنّي النمط نفسه؟ ما يمكن إعادة استخدامه مما تفعله Naia:

1. **بنية الدليل المزدوج** — `.agents/` (YAML/JSON مُحسَّن للذكاء الاصطناعي) + `.users/` (Markdown للبشر). الذكاء الاصطناعي يحصل على سياق فعّال من حيث الرموز، والبشر على وثائق سهلة القراءة.
2. **الترخيص المزدوج** — الكود Apache 2.0، والسياق CC-BY-SA 4.0. إبقاء سياق الذكاء الاصطناعي مفتوحًا عبر كل التفرّعات.
3. **ترويسة SPDX في كل ملف سياق** — تعريف ترخيص قابل للقراءة آليًا.
4. **قواعد حماية الترخيص في SoT** — يقرأ وكلاء الذكاء الاصطناعي القواعد ويفرضونها تلقائيًا.
5. **سيناريوهات الاختبار** — التحقق قبل الإصدار من أن وكلاء الذكاء الاصطناعي يرفضون فعلًا أي مخالفة.
6. **ملف CONTEXT-LICENSE** — تعريف واضح لنطاق تطبيق CC-BY-SA 4.0.

للاطّلاع على طريقة المشاركة راجع [دليل المساهمة](../CONTRIBUTING.md)، وللمواصفة التقنية الكاملة راجع [تفاصيل حماية الترخيص](../.users/context/contributing.md).

2. توليد Agent Core (Node.js، اتصال stdio)
3. عند إغلاق التطبيق يُغلَق فقط الـ Gateway المولَّد تلقائيًا

## نظام الذاكرة

تتذكّر Naia عبر الجلسات بفضل **Alpha Memory System** — بنية من 4 مخازن تحاكي الذاكرة البشرية:

| المخزن | المقابل | ما يخزّنه |
|-------|--------|----------------|
| **Episodes** | الحُصين | أدوار محادثة موسومة بطابع زمني |
| **Facts** | القشرة الحديثة | حقائق وتفضيلات وكيانات مستخرَجة |
| **Reflections** | العقد القاعدية | استراتيجيات مُتعلَّمة من إخفاقات سابقة |
| **Working Memory** | قشرة الفص الجبهي | السياق النشط للجلسة الحالية |

**كيف يعمل:**
1. تُسجَّل كل رسالة بدرجة وفق الأهمية (الحداثة، الصلة، الوزن العاطفي)
2. يُخزَّن المحتوى عالي الأهمية كـ **episode**
3. كل 30 دقيقة (أو بعد 5 دقائق من الخمول) تُدمَج الـ episodes في **facts** (يستعيد الذكاء الاصطناعي "ذلك اليوم")
4. عند بدء كل جلسة تُحقَن الـ facts والـ episodes ذات الصلة في السياق

**ذاكرتك ملكك:**
- مخزّنة محليًا في `~/.naia/memory/alpha-memory.json`
- لا تُرسَل إلى أي خادم — ولا حتى إلى Nextain
- استعرض واحذف الـ facts متى شئت من **الإعدادات ← الذاكرة**
- يكفي نسخ الملف لإكمال النسخ الاحتياطي

للاطّلاع على دليل المستخدم الكامل راجع [docs/memory.md](../docs/memory.md).

## تحديث نظام التشغيل

بُني Naia OS على [Bazzite](https://github.com/ublue-os/bazzite) (Fedora Atomic). التحديثات **ذرّية وآمنة**:

- **تلقائي**: تعكس إعادة البناء الأسبوعية أحدث تصحيحات أمان وتحديثات Bazzite
- **ذرّي**: تُنشَر الصورة الجديدة جنبًا إلى جنب مع الحالية — إن فشلت تبقى الصورة القائمة كما هي
- **التراجع**: اختر الإصدار السابق من قائمة GRUB للاستعادة الفورية
- **طبقتنا الإضافية**: تضيف فقط الحزم (fcitx5، الخطوط) + Naia Shell (Flatpak، صندوق رمل) + إعدادات العلامة — ولا تمسّ أبدًا النواة أو محمّل الإقلاع أو نواة systemd

```
Bazzite base update → Weekly auto-rebuild → Container smoke test → ISO rebuild → R2 upload
                                                                 ↘ GHCR push → user bootc update
```

تفاصيل خط أنابيب التحديث: [`.agents/context/update-pipeline.yaml`](../.agents/context/update-pipeline.yaml)

## عملية التطوير

### تطوير المزايا (الافتراضي) — تطوير قائم على القضايا (Issue)

```
ISSUE → UNDERSTAND → SCOPE → INVESTIGATE → PLAN → BUILD → REVIEW → E2E → SYNC → COMMIT
```

- **3 بوّابات إلزامية** — تأكيد المستخدم مطلوب عند UNDERSTAND و SCOPE و PLAN
- **بعد الموافقة على الخطة** — ينفّذ الذكاء الاصطناعي من BUILD إلى COMMIT متواصلًا بلا توقّف
- **المبدأ** — اقرأ شيفرة المنبع أولًا (لا تخمين). أدنى تغيير. لا تكسر أبدًا كودًا يعمل.
- **الالتزام (Commit)** — بالإنجليزية، `<type>(<scope>): <description>`
- **المنسّق** — Biome (مسافات جدولة، علامات اقتباس مزدوجة، فواصل منقوطة)

## مفتوح المصدر بطبيعته الذكية (AI-Native)

في عامَي 2025–2026 كانت معظم مشاريع المصدر المفتوح *تتحصّن* ضد مساهمات الذكاء الاصطناعي. **تتّخذ Naia النهج المعاكس تمامًا**: نصمّم المشروع بحيث تكون المساهمة بمساعدة الذكاء الاصطناعي عالية الجودة افتراضيًا.

> **"لا تتحصّن ضد الذكاء الاصطناعي، بل صمّم معه."**

### كيف يعمل

```
Person (any language) → AI → Git (English) → AI → Person (any language)
```

- **اكتب القضايا وطلبات الدمج بلغتك** — يترجمها الذكاء الاصطناعي كلّها
- **المساهمون والمشرفون كذلك يستخدمون الذكاء الاصطناعي** — في البرمجة والمراجعة والفرز (triage)
- **سياق `.agents/` الغني** يعمّق فهم الذكاء الاصطناعي للمشروع — وكلّما تحسّن فهم الذكاء الاصطناعي ارتفعت جودة المساهمة
- **10 أنواع مساهمة** — ترجمة، مهارات، ميزات، أخطاء، كود، وثائق، اختبار، تصميم، أمان، سياق
- **سجلّات العمل بلغتك الأم** — احتفظ بمستودع خاص بلغتك؛ وتُراجَع سجلّات Git عبر ترجمة الذكاء الاصطناعي

هذه ليست مجرّد سياسة. إنها بنية معمارية. دليل `.agents/` ووثائق المرآة الثلاثية وقواعد حماية الترخيص كلها مصمّمة لجعل تعاون الذكاء الاصطناعي بنيةً لا صدفة.
